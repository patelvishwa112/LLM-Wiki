# Bezos Writing Framework — 6-page narrative memos (Dickie Bush)

## Key Takeaways

- Amazon replaced PowerPoint with **six-page narrative memos**. Bezos: there is no way to write a narratively structured six-pager without clear thinking.
- **Memo skeleton:** Introduction → Goals → Tenets → Current state → Lessons learned → Strategy.
- Meetings open with **~20 minutes of silent reading** so everyone shares the same context before discussion.
- **Six writing rules:** (1) "So what?" test on every point, (2) <30 words per sentence, (3) avoid jargon/acronyms (spell out on first use), (4) replace adjectives with data, (5) subject-verb-object sentences, (6) ban weasel words ("nearly," "significantly," "in order to" → "to").
- Quality cost: great memos take **days** — write, rewrite, peer feedback, rest, edit with fresh eyes.
- Culture goals: clear thinking, fewer low-signal meetings, frictionless knowledge transfer across teams.
- Practice path: pick 1–2 rules, daily writing time, low-stakes (e.g. Slack), feedback loops; optional 6-pager for the next high-stakes meeting.

## Why it matters

Writing-as-thinking is a durable ops pattern, not just Amazon lore. The silent-read + narrative format is a high-leverage alternative to slide decks that hide fuzzy reasoning. The six rules are portable style constraints for any knowledge work (PRDs, design docs, investment memos, agent system briefs).

## Skeptical read

Dickie Bush packages a well-known Bezos/Amazon practice as a "steal this framework" thread that funnels into a free long-form content guide and daily-X habit pitch. The core memo structure and rules are real and useful; treat the creator-economy CTA as optional marketing, not part of the Amazon system.
