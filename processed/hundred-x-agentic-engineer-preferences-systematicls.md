# 100x agentic engineer (Fable & GPT 5.6 era)

sysls (@systematicls) — systematic quant / @openforage — argues **agent capability has outpaced human specification**: models (~GPT 5.5+) can implement almost anything; productivity variance is again about the **human behind the agent**, not flattening to one speed.

## Core thesis

- Classic **100x engineer** returns in **agentic** form — skill ceiling resembles handwritten-code mastery  
- **No free lunch**: agents pick points on a huge Pareto frontier using incomplete prefs + their own assumptions → “stupid” outputs are often **trade-off mismatch**  
- Great agentic engineering = **know what you want** at each abstraction layer (architecture, business, ops)

## Preference vocabulary

| Axis | Declarative | Imperative |
|------|-------------|------------|
| Focus | **What** outcome | **How** path |
| Pro | Generic principles, scales with smarter models | Opinionated workflows, firm standards |
| Con | Agent fills gaps → wrong bounds | Locks in bad opinions |

**Strategic** (firm-wide, rules/skills, survive compaction) vs **tactical** (project-specific; often **20% imperative / 80% declarative** on critical vs rest).

Encode security/throughput principles declaratively; non-negotiable deploy/audit flows imperatively (auditor + red-team loop example).

## Taste & prompts

- Use agents for **Socratic** preference discovery on high-leverage assumptions  
- Prompts still matter: disambiguate interpretations; **don’t pollute** with future project B when building A — generalize interfaces instead  
- Clear preferences → **loop objectives** agents can hill-climb  
- Future question: not “can you build it?” but **“do you know what you want built?”**
