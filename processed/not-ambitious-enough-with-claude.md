# You're not ambitious enough with Claude

Christine Zhu (platform PM) describes moving past **dopamine-backlog automation** into **high-leverage judgment work** once Cowork/Fable and context depth caught up.

## Shreyas three levels → Claude roles

| Level | Role for Claude | Tactics |
|-------|-----------------|---------|
| **Optics** | Autopilot | Twice-weekly Cowork scheduled task: Slack, calendar, notes, git → Google Sheet status board + packaging skills; review/edit only |
| **Execution** | Copilot | Weekly context-dump skill (coach-like patterns, leverage %), Jira/roadmap planning skill, monthly support-theme analysis — **ask for judgment**, not summaries |
| **Impact** | Sparring partner | Onboard context folder (decks, roadmap, code summary, stakeholders, goals, style guide); voice-dump in 10 minutes; Fable high/max for first drafts; HTML artifacts for thinking, Figma connector for shareable visuals |

## Core claim

**Judgment is mostly a function of context.** A platform PM routing 5–10 workstreams cannot hold as much written context as Claude with connectors; the bottleneck shifts to feeding evergreen context and raising ambition on impact-tier tasks—not reserving “defensibly human” work by default.

## Secondary loops

- **Claude Code** for technical fluency, architecture docs, PRD debate, small PRs.
- **Rolling context-dump history** as career coach for blindspots and review writing.
- **Setup trap warning:** perfect tooling can become another dopamine backlog.
