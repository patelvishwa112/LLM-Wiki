# Step-By-Step LLM Engineering Projects (2026 Edition)

By @TheAhmadOsman | May 24, 2026 | 113K views | 1.6K bookmarks

## Key Takeaways

- **34 projects across 21 parts** covering the entire LLM stack: tokenizer → embeddings → attention → transformer → training → decoding → KV cache → MoE → data → scaling → post-training → quantization → serving → eval → RAG → agents → multimodal → interpretability → safety
- **Core methodology**: Build → Plot → Break → Explain. Implement from scratch, ablate key components, document failures
- **12-week lock-in plan** from tokenization through full capstone model delivery
- Directly references Anthropic work: [[sparse-autoencoders]], [[constitutional-ai]], mechanistic interpretability
- Philosophy: "Fundamentals remove the hiding places" — frameworks and wrappers only make sense after touching the model directly

## Summary

A project-based roadmap for turning LLM fundamentals into working systems. Each project teaches one hard-earned concept through implementation, ablation, and documentation. Covers 34 projects: tokenizer from scratch, attention hand-wiring, KV cache, MQA/GQA/MLA, MoE routing, state-space models, diffusion LMs, data pipelines, RLHF/GRPO/RLVR, quantization, serving, evaluation harnesses, RAG, agents, and a full capstone.

Notable: explicitly recommends studying Anthropic's sparse autoencoder and Constitutional AI work for the interpretability and safety sections.
