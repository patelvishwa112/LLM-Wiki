## Key Takeaways

**The Night Shift mental model:** Your second brain is an intelligence agency with a day shift (you capture) and a night shift (300-agent swarm processes while you sleep). The swarm has five distinct roles: Scouts (fetch full text), Catalogers (atomic notes), Cartographers (link graph), Critics (flag contradictions with [FRICTION] blocks), Editors (synthesis + morning briefing).

**Pipeline architecture (refinery, not drawer):**
- 0-raw/ — sacred intake (never edited)
- 1-desk/ — transient scratch space (cleared nightly)
- 2-atoms/ — one idea per file, permanent core (Zettelkasten principle)
- 3-threads/ — living synthesis documents that grow automatically
- sources/ — read-only originals (ground truth)
- briefings/ — what the night shift writes back to you
- playbooks/ — skill files (job descriptions)
- house-rules.md — operating charter with Prime Directive ("No source, no note")

**Four staged playbooks (cron-scheduled):**
1. Scout Run (11pm) — WebBridge + browser to fetch full articles
2. Refinery Run (3am) — Agent Swarm (one sub-agent per source, parallel) does atomic splitting, linking, friction detection
3. Editor Run (6am) — update threads + write briefing with contradictions highlighted
4. Audit Run (Sunday) — integrity sweep, flags orphans, stale threads, Prime Directive breaches

**Guardrails (non-negotiable):**
- Prime Directive: every atom traces to real source (prevents confident hallucination)
- ask-before-acting safeguard on while learning
- git version control from day one (rollback seatbelt)
- weekly friction list review (the only human work left)

**Compounding effects:**
- Filing/linking/sorting stops being your job
- Critics start arguing with you (surface contradictions you forgot)
- Threads become smarter than your recall after months

**MCP integration:** Expose vault as MCP server so Kimi Work, coding agents, editors, and any MCP tool share the same memory.
