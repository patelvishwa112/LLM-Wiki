# Four-layer agent memory stack

**@MatthewGunnin** field guide from **6 months** running **Ella (OpenClaw)** and **Lyra (Hermes)** with shared continuity.

## Layers

| Layer | Role |
|-------|------|
| **1 In-session** | Editable identity markdown + memory **index** (TOC → per-fact files); index always in context |
| **2 Hindsight** | Local post-session fact banks; query on startup; human gate before promoting to index |
| **3 Nexus** | Shared Obsidian vault—append-only **live-context**, project state, decisions log, working checkpoints |
| **4 gbrain** | MCP search over vault—semantic/recall without stuffing full repo into context |

## Coordination invariant

Append-only signed log lines; read-before-reply; explicit acknowledgment of peer entries; decisions log for durable cross-agent truth.

## Start small

Identity + index + shared live-context file (weekend); automate retention after manual trust.

## Honest tradeoff

Markdown discipline, not embedding magic—maintain and review what becomes permanent.

## Why it matters

Operational blueprint linking vault notes on **gbrain**, **OKF/Obsidian**, Hermes/OpenClaw harnesses, and multi-agent memory landscape.
