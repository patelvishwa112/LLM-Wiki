# Agents as employees — six environment shifts

zodchiii argues **harness UX** beats model IQ for “coworker” agents: the boring 90% (wait, ask, approve, remember, coordinate).

## Checklist (product-agnostic)

| # | Shift | Mechanism |
|---|--------|-----------|
| 1 | Wait like people | Question/queue/connection/approval **cards**, not silent spinners |
| 2 | Ask before guessing | Structured unblock prompts in UI |
| 3 | Sign-off on risk | Email, publish, secrets, external writes |
| 4 | Team coordination | Coordinator, visible task tree, isolated threads, durable state |
| 5 | Own outcomes | Cron/scheduled **responsibilities** across sessions |
| 6 | Live somewhere reliable | Recovery + contracts (author: DIY VPS vs hosted ops) |

Maps cleanly to Hermes patterns: **clarify**, **cronjob**, **kanban/delegation**, **Signal approvals**, **profiles/memory**—without needing a SaaS wrapper.

## Skeptical read

Long **Teamly** ad. Useful as **vocabulary** for product/agent builders; verify infra claims independently.

## Why it matters

Same thesis as loop engineering + human-in-the-loop notes: chat UI is the wrong shell for **delegated work**.
