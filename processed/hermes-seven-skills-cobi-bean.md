# Seven Skills for a Serious Hermes Agent

By @cobi_bean.

## Core rule

**Repeat a procedure more than twice → make it a skill.** Skills are the highest-leverage part of Hermes: they turn operating procedure into **automatic environment**, not chat reminders.

## The seven

| # | Skill / tool | Role |
|---|----------------|------|
| 1 | Superpowers (brainstorming) | Design before implementation; approval before build |
| 2 | Soul-Grader-Skill | Enforce `soul.md` — mission, boundaries, truthfulness |
| 3 | Project-Memory-Skill | Append-only repo memory under `docs/memory/` |
| 4 | Agent-Memory-Wiki | Obsidian vault logs + janitor; curated handoffs |
| 5 | gogcli + wrapper skill | Safe Google Workspace CLI with approval gates |
| 6 | Hermeshub Diagram Maker | Mermaid from NL; structure must be visible |
| 7 | Hermeshub Skill Factory | Meta: repeated failures → new skills |

## Why it matters

Big prompts and tool lists are not enough. Durable agents need **habits**: how they start work, persist memory, access external systems safely, diagram systems, and **compound** by crystallizing corrections into skills.

Maps directly to Hermes patterns: `skill_manage`, memory vs skills split, Obsidian wiki ingestion, and goal/autonomous modes that assume skills are loaded first.
