# Claude Personal Assistant — $0 Build

## Key Takeaways

1. **Everything inside one Claude Project.** No external tools, no API costs, no automation platforms. The Project is the assistant's permanent home — system prompt + context files + persistent memory.

2. **Context files are the identity layer.** Three files (about_me.txt, preferences.txt, goals.txt) turn a generic chatbot into a personal assistant. Combined with Claude's Memory, the assistant compounds sharper every week.

3. **Five components, zero new spend.** Project (home base) → Master Prompt (brain) → Context Files (identity) → Web Search (senses) → Artifacts (hands). All included in Claude subscription.

4. **Daily modes make it practical.** "brief me", "plan my day", "research [X]", "build me [X]" — simple commands that leverage persistent context for compounding value.

5. **Artifacts are the underused superpower.** One-sentence requests yield functional tools: habit trackers, planners, calculators, decision matrices. The assistant has hands, not just a mouth.

## Why This Matters

This is the same pattern appearing across the agent ecosystem: persistent context + reusable skills + tight feedback loops. The $0 Claude assistant is the consumer-grade version of the agent harness being built at the enterprise level (see [[wtf-is-a-loop]], [[harness-is-the-product-context-aware-agents]]). The architecture is the same — what differs is scale.
