# /opportunity — AI PM OS workflow (Torres OST)

George (prodmgmt.world) on **mapping opportunities before prioritizing features**—where strategy actually lives.

## Core argument

**RICE** and numeric ranking assume you already have a solution shape. One level up, decimals create false certainty and kill the conversation that should compare bets.

Torres framing: pick **which customer needs to explore**, not which feature to ship first.

## Three-step chain

| Step | Skill | What it enforces |
|------|--------|------------------|
| 1 Intake | Shape inputs | Outcome = behavior metric; journey = **time moments**, not eng categories |
| 2 Tree | OST build | Needs not solutions; distinctness + parent-child tests; optional MECE; evidence-linked nodes |
| 3 Select | Four lenses | Sizing, market, company, customer (importance × dissatisfaction); **forced comparison** in prose |

Selection rules: defer effort; reversible commitment; iterate tree every few interviews.

## Where AI helps

Not magic synthesis speed alone—**method enforcement** under load (reframe, dedupe, structural checks humans skip under deadline). Parallels **skills/workflows** pattern in Claude Code / PM OS.

## Why it matters

Pairs with vault notes on **problem-first**, **testing business ideas**, and **evidence-first PRDs**—same discovery stack, different choke point (opportunity map vs assumption tests vs ambient agents).
