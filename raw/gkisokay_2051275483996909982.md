## Post

> It's the agent most builders skip. It turns the outside world into compounding intelligence. In 3 months, it's logged 8,000+ pieces of evidence across 16 topics, and every other agent in my stack starts smarter as a result.
>
> It's the research agent. The always-on research department inside Hermes or OpenClaw. It watches what is happening, reads my own workspace, tracks what is moving, and writes everything into a local vault that every other agent in my stack can reuse.
>
> It is not a scraper. This isn't a daily summary bot. This part of the system ensures tomorrow's agents don't start with yesterday's knowledge.
>
> The first version of a research agent is usually a feed reader. The better version is a librarian. The best version is an evidence operator.
>
> Skip that distinction, and research bots become hallucination laundries: confident prose with no separation between what was observed, what was claimed, and what was actually verified.

## Why This Matters

A typical research agent scrapes, summarizes, and posts a digest. Then the next run starts almost from scratch, except for a note.

That looks productive because there is output everywhere. But output is not the same thing as compounding evidence.

A real research agent should be able to answer:
- What do we know now that we did not know before?
- Which claims are strong, and which are interesting but under-evidenced?
- Which sources and topics keep being useful?
- Which signals belong to which agent, and which old beliefs are now stale?

## What The Vault Actually Holds

Graeme's live vault after 3 months:

- 2,694 findings (individual observed signals)
- 2,694 source evidence records
- 13 indexed dossiers (living topic files)
- 18 registered source surfaces
- 8 operational modes
- 6 handoff lanes routing to other agents
- 36 vault tools
- 6-hour refresh cadence + 3 daily delivery jobs

## The Chain (Critical)

Raw input → finding → claim → verified knowledge → task → approved work

"A finding is not a claim. A claim is not verified knowledge. A source record is not a conclusion. A dossier is not a daily summary. A weak signal is not a task."

The research agent works because it has space for each stage, rather than flattening everything into confident prose.

## Architecture

Graeme's agent stack:
- Research agent = evidence collector
- Main = conscious operator
- Subconscious = pattern-noticer
- Coder = builder
- QA = auditor
- Content = publishing operator

Everything is downstream from the research agent. It contributes evidence and implications. It does not own the whole machine.

"When research, judgment, building, and publishing collapse into a single agent, the system starts treating every interesting signal as an action item."

## The SOUL.md

The file that defines the research agent's role — job description, identity, and boundary layer. The loop: observe → infer priorities → gather evidence → deepen one question → update vault → route implications.

"Reading the news creates summaries. Running a research loop creates judgment."

## The Vault Structure

Minimum recommended shape:
```
research-vault/
├── raw/              — unprocessed capture layer
├── knowledge/        — claims, findings, sources
│   ├── claims/
│   ├── findings/
│   └── sources/
├── dossiers/         — living topic files
├── verification/     — under-evidenced claims
├── summaries/        — daily, weekly digests
├── handoffs/         — routing to other agents
├── decisions/        — what was decided, by whom, on what evidence
├── runs/             — each refresh leaves a receipt
├── ops/              — source balance, health checks
├── health/           — broken links, missing front matter, gaps
├── context/          — interest profile, source plan
└── wiki/             — Obsidian-compatible compiled pages
```

## The Interest Profile

The research agent doesn't ask "what is trending?" It asks "What does the user care about now?"

Rebuilt from: shared Hermes state, durable notes, thesis files, recent posting behaviour, shipped builds, repeated questions, prior research outputs.

Files: `research-vault/context/interest-profile.json`, `research-vault/context/interest-profile.md`

"If your agent does not know what matters to you, it will optimize for whatever is loudest."

## The Source Plan

Rule: Prefer sources that change decisions.

"If your research agent collects everything, it will drown. If it only collects what is easy, it will overfit to the most convenient API. The source plan is where the agent learns taste."

## Dossiers, Ledgers, and Verification

Dossiers hold: why the topic matters, current signal, source trail, contradictions, open questions, product/build/content/monetization implications.

The research agent starts asking:
- What can we decide?
- What can Hermes build?
- What can I explain?
- What can become a product?
- What should we not trust yet?

Verification queue: "The research agent needs somewhere to put uncertainty. Without a verification queue, uncertainty gets ignored or over-promoted. With a queue, the agent can say, 'This might matter, but do not build on it yet.'"

## Operator Surfaces

- Operator brief: what changed, what deserves attention, what's blocked by weak evidence, what should other agents do
- Cockpit: for scanning
- Action ledger: for follow-through
- Focus message: what matters now
- Dispatch JSON: for routing

"A research system should make the next decision easier."

## Quality Gates

- Source balance: is the run over-dependent on low-trust surfaces? stale collectors? degraded lanes?
- Health checks: broken links, missing front matter, gaps in source trail, orphan candidates, stale reviews, verification pressure, wiki compile drift

"If the vault is structurally unhealthy, do not trust new synthesis until the structure is fixed."

## Runtime

Cron jobs at `research-agent/cron/jobs.json`:
- Daily summary → research channel
- Subconscious brief → local
- Midday focus → research channel (rebuilds cockpit from existing artifacts, does not scrape)

Model separation: cheaper/local models for routine refresh, stronger models for synthesis, judgment, handoff quality.

"Research is many jobs wearing one coat."

## Guardrails

Not allowed: trading decisions, public posts, purchases, partnerships, secrets/auth, turning weak signals into approved tasks, pretending stale data is fresh.

Allowed: read shared context, collect evidence, score source quality, write dossiers and operator briefs, maintain verification queue, route implications to other agents, surface degraded collectors.

"The research agent can influence the machine, but it cannot seize the steering wheel."

## The Real Lesson

"A real research agent knows what you care about, what it has seen before, which sources are strong, which collectors are degraded, and which claims are not yet ready. Most importantly, it knows when to say, 'This is interesting, but not safe to build on yet.' That is where research agents become valuable — they can preserve uncertainty without losing momentum."

## If You Want To Build One

1. Give it a vault and an identity
2. Source plan and knowledge ledgers
3. Verification queue and handoff lanes
4. Health checks and a schedule
5. Then let other agents consume its outputs

"A normal agent answers the prompt in front of it. A better agent remembers what happened. A research agent builds the evidence base that makes future prompts smarter before they are even asked."
