# How to Use RLMs in Deep Agents

**Author:** Sydney Runkle (@sydneyrunkle)  
**Posted:** Jul 1, 2026

Context rot degrades agents as context grows. Recursive language models (RLMs), from Alex Zhang et al. (MIT CSAIL), load the prompt as a REPL variable and use code to peek, decompose, and recursively call over snippets—instead of turn-by-turn lossy summarization.

Example: average deal size across 10,000 call transcripts—turn-by-turn running totals drift; RLM-style orchestration keeps counting in code.

Paper claim: process inputs ~two orders of magnitude beyond context window, beating vanilla agents. LangChain added RLM-style support in **Deep Agents** via dynamic subagents + code interpreter.

## Case for RLMs

RLMs recursively call themselves or other LLMs before final answer. REPL holds full prompt as variable; code chunks and maps recursively.

Subagents already isolate context, but tool-based orchestration breaks at hundreds of calls, branching, multi-phase work.

Programmatic subagent dispatch enables:
- **Deterministic coverage** — `for b in batches` touches every batch by construction
- **Bespoke orchestration** — parallel/sequential/branching pipelines as code

## Deep Agents implementation

Model writes short script; **CodeInterpreterMiddleware** (QuickJS) executes it. Example: `Promise.all(pages.map(page => task({...})))`.

Terminology note: paper loads entire prompt into interpreter with plain LM recursive calls; Deep Agents ships **recursive agents (RA)**—subagents with tools/state dispatched from code. RLM paper motivated the architecture.

Common code patterns: chunk context + recursive LM calls; aligns with Claude Code dynamic workflow patterns (fan-out/synthesize, classify/act, adversarial verify, generate/filter, tournament, loop-until-done).

Orchestrator and subagents can mix models (e.g. frontier orchestrator + open-weight subagents).

## OOLONG benchmark (AgNews)

Long-context reasoning / aggregation over nearly every row. Thousands of headlines with date/user; agent classifies into four topics and aggregates.

Question types: counting (all rows), user-filtered counts, temporal filters.

Smoke test (not fully optimized):

| Context | Without REPL | With REPL |
|---------|--------------|-----------|
| 64k | 0.58 | 0.67 |
| 128k | 0.44 | 0.79 |

At 128k plain agent often refuses or says blocked; RLM-enabled agent completes. REPL path slower, higher output-token cost.

Scoring: OOLONG exact match / partial numeric credit.

## Get started

```bash
pip install -U "deepagents[quickjs]"
```

```python
from deepagents import create_deep_agent
from langchain_quickjs import CodeInterpreterMiddleware

agent = create_deep_agent(
    model="openai:gpt-5.5",
    middleware=[CodeInterpreterMiddleware()],
)
```

Prompt with **"workflow"** to trigger dynamic subagents. **dcode** CLI ships interpreter pre-enabled.
