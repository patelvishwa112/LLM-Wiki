## Basic LLMs

Skipped — covered by numerous excellent explainers (3Blue1Brown recommended).

## VLMs & Omni Models

Key difference: VLMs process multimodal inputs + text. Need to translate input into same embedding space as text and add training data. Three mediums: images, video, audio. Many frontier models (including Opus 4.8) may only have image support with no native audio/video encoders.

### Image Encoders

Naive approaches: encode every pixel (too sparse, context window issues) or text-transcode first (too compressed, pre-commits to representation). Solution: latent representation translated into text subspace.

**CLIP (OpenAI, early 2021):** Dual encoder — image + caption into shared subspace.
**SigLIP (Google):** Improved CLIP. SigLIP-2 is state of art, used in Qwen 3.5. Called a ViT (Vision Transformer).

Counterintuitive finding: dividing image into non-overlapping 16x16 grid patches works well. An Image is Worth 16x16 Words paper. Projector compresses further (256→128 tokens).

Training recipe: pre-trained SigLIP checkpoint + base LLM → warm up projector → fine-tune encoder on domain data → end-to-end multimodal training with low LR on encoder.

### Video Encoders

Naive: every frame through ViT with temporal embedding — ridiculously inefficient. Strategies: sample every N frames, smart frame-jump detection, encode only diffs between full-frame samples. Active research area, crucial for world models.

### Audio Encoders

Borrows from Whisper (OpenAI, 2022). Raw audio → 25ms sliding window with 10ms hop → FFT → mel-spectrogram → audio encoder (AuT in Qwen) → projector → LLM embedding space.

Trained as encoder-decoder: audio → text → audio. Must learn beyond transcription — pitch, emotion, music, mechanical sounds. Audio understanding data (description of how audio sounds) added to mix.

### Training and Shared Representations

Multimodal training is more a data problem than architecture problem. Computer use data is great fit: tractable, easy to grade, image inputs. Open question: are screenshots a different distribution from natural images?

Ideal: shared representation across modalities teaches physics from audio/video. Reality: transfer learning poorly understood, multimodal crowding can cause regressions.

Gemma 4 12B: stripped-down encoders, trained end-to-end from scratch — trend toward minimal preprocessing.

## Mixture of Transformers (MoT)

Meta, late 2024. Split transformer body into N parallel weight stacks (one per modality), but share one global self-attention. Text tokens flow through text weights, video through video weights, but attention spans all tokens. No token-level routing — each modality deterministically uses its dedicated stack.

## Realtime and Interaction Models

### Realtime Audio Models

Original: 4-model pipeline (VAD → ASR → small LLM → TTS). Problems: brittle scaffolding, discrete turns, high latency.

**Improvement 1 — Native audio I/O:** 80ms audio → 8 discrete tokens (semantic + acoustic). Discrete representation sufficient for speech, not music.

**Moshi (open-source speech-to-speech):** Temporal autoregressive model → 1 token per 80ms → depth model converts to 8 discrete audio tokens via RVQ. Duplex: two concurrent streams (model output + user input), 80ms blocks, constant running. Inner monologue: 17 tokens/turn (8 output audio + 8 input audio + 1 text for reasoning).

### Async Tool Calling

Model outputs audio + text streams simultaneously. Text stream makes tool calls to background agent. Background processes on own time, inserts into "oracle stream." Progressive results with varying confidence. Model learns to stall while waiting.

### Interaction Models (Thinking Machines)

Native (video, audio, text) → (audio, text). Minimal preprocessing, no pre-trained components. Trend toward less opinionated encoders.

Overall vision: latency-sensitive interaction model sitting on top of powerful VLM systems.

## Image and Video Generation

### Early Image Models

Diffusion: inject noise → model denoises using prompt → recover original. Inference: arbitrary prompt + noise → new image. Flow matching: learn velocity field transporting noise→data. Rectified flow: straight paths, fewer steps.

**DiT (Diffusion Transformer):** Breaks image into patches, attends via standard transformer, diffusion head generates direction vector per patch, ~20 steps. Bookended with VAEs.

### Autoregressive Image Models

VQ-VAE discrete patch tokenizer + autoregressive generation. Struggle: vocabulary too coarse for fidelity. GPT-4o hybrid: route to text tokens OR diffusion head for continuous patches.

### The Image Harness

Beyond pure forward pass: image-to-image editing, infill models, python copy-paste tools, thinking-step prompt rewriting, sketch-before-generate.

### Video Generation Models

Two problems: generation speed (hundreds of frames) + object consistency. Dominant approach: DiT + flow matching generating 3D blocks (16x16 patch × 4 frames). Ideas for attending to more history: compressed video history, tight frame buffer + sparsification, low-res upsample, learned latents.

**Frame-conditioned video models:** F(Text, List[Image]) → Video. Used for character-consistent generation.

## World Models

**Core idea:** F(Previous Frame, Action) → Current Frame. Autoregressively explore world by taking actions and observing new info.

**Constraints vs video models:** Must predict causally forward (not attend bidirectionally), must be very fast (near-realtime), can sacrifice pixel quality for task-relevant objects.

**Use cases:** train robotic policies in sophisticated simulators, forecast actions for trajectory search, auxiliary loss for training.

### Latent Space vs. Pixel Space

**Pixel-to-pixel:** VAE encode → DiT + flow matching → decode. Objective: reconstruct full image pixel-for-pixel.

**Latent space (JEPA/LeCun):** ViT encodes frames into latent → small transformer predicts next frame in latent. Loss stays out of pixel space → more data efficient but can collapse. Solutions developed to mitigate.

### Training and Action Inference

Multi-stage: first learn physics/dynamics (no action conditioning), then add action conditioning with robotics data, finally fine-tune to deployed instance. Action inference: generate synthetic off-policy data, MCTS planning over plausible paths, or reinforcement learning (Dreamer).

## VLAs (Vision-Language-Action Models)

**RT-2 (Google DeepMind, 2023):** Discretized robot actions as tokens in same vocabulary as text. Co-fine-tuned on web data + robot demos. Most modern VLAs use Action Head (vector encoding delta position) instead, output no text.

### History of Optimizing VLAs

1. Autoregressive awkward for interdependent spatial coordinates → diffusion head (flow matching)
2. Speed: predict 10-20 future frames, combine trajectories for smoothness
3. RL for trajectory accuracy on fine motor skills (Physical Intelligence)
4. Long-term coherence: MEM (PI) — two-layered system with memory string + subtask decomposition; compressed video encoding for short-term memory
5. MemoryVLA: latent snapshots across text+images, auto-compressed
6. Cross-embodiment: PI forces 32-length action vector, masks unused slots. Human egocentric data improves all embodiments.

### Mixture of Transformers for VLAs

**Cosmos 3 (NVIDIA):** Two towers — reasoner (autoregressive VLM) + generator (diffusion model). Shared attention, separate feed-forward weights. Shared 3D multimodal rotary positional embedding. Flow: reasoner produces understanding tokens → generator denoises video/audio/action tokens in parallel.

## Biological Foundation Models

### DNA Sequence Models

**Evo2 (Arc Institute):** Modified transformer, 4 tokens (A,C,G,T), 1M context. Uses short/medium/long convolutional filters + gates interspersed with attention instead of full attention.

### Protein Models

**ESM 2 (Meta):** BERT-for-proteins. Attention relationships embed 3D structure — if attention head links two tokens, they're close in 3D space. Used to build ESMFold.

**ESM 3:** Predicts sequence, structure, and function. VQ-VAE → 4096 discrete tokens for position. Up to 8 function tags per position (e.g., "ATP binding"). Bidirectional transformer with geometric structure embedding.

**AlphaFold 2 (DeepMind, 2021):** Nobel Prize. MSA (multiple sequence alignment) + EvoFormer (48 iterations of co-evolution between MSA and pair representation). Triangle inequality enforced. Structure model generates coordinates → converted back to pair representation → outer loop 3×.

**AlphaFold 3:** Extends to DNA/RNA/small molecules + multi-structure interactions. PairFormer replaces EvoFormer, diffusion replaces structure model.

**Hot frontier:** Full cell models (Chan Zuckerberg, DeepMind, Arc).

## Glossary (Key Terms)

- **ViT:** Vision Transformer — encoder chunking image into patches
- **DiT:** Diffusion Transformer — denoising head instead of next-token
- **VAE:** Encoder-decoder with sampleable latent
- **VQ-VAE:** VAE where latent snaps to fixed vocabulary
- **MoE:** Mixture of Experts — sparse activation of subnetworks
- **MoT:** Mixture of Transformers — modality-split transformer stacks
- **Diffusion:** Denoise noise → generate samples
- **Flow Matching:** Direct noise→data path, fewer steps
- **MCTS:** Monte Carlo Tree Search planning
- **JEPA:** Latent-space next-state prediction
- **MSA:** Multiple Sequence Alignment for proteins
- **Moshi:** Open-source duplex speech-to-speech
