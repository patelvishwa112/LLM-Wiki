## Post

> TL;DR: We extended Miles with a multi-adapter LoRA training path that lets us train thousands of LoRA adapters concurrently and asynchronously. The core change is a modification to Megatron-Bridge + Miles which allows us to load multiple LoRA adapters as a single matrix. On Qwen3.6-35B-A3B + GSM8K stress test, we ran 1,536 LoRA adapter instances concurrently with step time under 3 minutes.

## Overview

LoRA-based post-training decomposes a model into two components: a shared base model and a lightweight low-rank adapter. This works well for single policy training, but underutilizes resources when many policies are fine-tuned in parallel — each run still deploys the full base model alongside its adapter, replicating the same weights across concurrent runs.

"What if we could share the same base model between policies, and just fine-tune different LoRA adapters in a single batch?"

Built on prior work supporting LoRA RL for Qwen3.5. Extended from "one LoRA policy" to "many LoRA policies, in the same training step."

Motivation: base models are large, LoRA adapters are small. To run thousands of RL experiments (prompt/harness, reward design, curriculum ablations), you can't replicate the full base model for every individual training run.

## Results

Stress test: GSM8K training dataset, Qwen3.6-35B-A3B, up to 1,536 concurrent LoRA adapter instances.
- Training step time: under 3 minutes (inference ~2 min, training ~1 min)
- Reference model and policy model show virtually no divergence (consistent with every policy being a small LoRA delta off the same base model)

Trade-offs: deploying 128 LoRAs simultaneously has performance implications. Ablated across multiple adapter-LoRA rank configurations using Qwen3-4B and Prime Intellect's Reverse-Text-RL dataset.

## Benefits Beyond Resource Efficiency

1. Online loading/unloading with multi-LoRA bypasses cold-start time for spinning up large base models on a cluster → faster iteration
2. Multi-LoRA training can quickly scale up post-trained expert models for on-policy distillation (increasingly popular post-training technique)
