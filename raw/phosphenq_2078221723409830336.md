# Context engineering field guide — @phosphenq

Posted: 2026-07-17
URL: https://x.com/phosphenq/status/2078221723409830336

Everyone on your timeline started saying "context engineering" instead of "prompt engineering," and almost no one can tell you what changed.

Here is the whole thing in plain language: what it is, why a bigger context window makes your answers worse, and the 12 moves that decide every reply you get.

Most people who use AI every day still think the magic is in the prompt. The clever wording, the role-play, "act as a senior engineer," "take a deep breath and think step by step."

9 out of 10 have never touched the thing that actually decides the answer: everything else in the context window at the moment they hit enter. What the model was told, what it remembered, what it retrieved, what examples it saw.

No system design, no retrieval, no memory, no compaction. Just a person typing a nicer sentence into a box and hoping.

This is the field guide to context engineering, the skill that quietly replaced prompting. Sourced from Andrej Karpathy and Tobi Lütke, from Anthropic's engineering team, and from research on how a model actually reads a long window.

## PART 1 · What changed, and why bigger is worse

### 01. The skill you were using had a bigger name

A prompt is one message. That is over. A modern model answers out of a whole window: system instructions, history, retrieved documents, examples, tools, memory. The prompt was the tip. The window was the iceberg.

Karpathy (June 2025): "context engineering is the delicate art and science of filling the context window with just the right information for the next step."

Tobi Lütke: "the art of providing all the context for the task to be plausibly solvable by the LLM."

Prompt engineering is writing one good sentence. Context engineering is designing the whole window that sentence lives inside.

### 02. Prompting didn't die. It got swallowed

Models outgrew ceremony. Bottleneck moved upstream to multi-turn work with retrieval, tools, memory. One-shot prompt stopped being the unit when agents run hundreds of steps.

Anthropic: context engineering is "the natural progression of prompt engineering."

Honest gate: one-shot chatbot Q&A still wins with a good sentence. Context engineering matters when sessions get long, work spans steps, or answers matter.

### 03. Why a bigger window makes it worse

Lost in the Middle (Stanford 2023): performance highest when relevant info is at beginning or end; degrades in the middle. Attention is a U-curve.

Chroma (July 2025), 18 frontier models: performance grows increasingly unreliable as input length grows. Drop starts before the window is "full." Tidy coherent context hurt attention more than shuffled text. Long context bills accuracy and money.

Anthropic: attention budget; context is "a finite resource with diminishing marginal returns." Job is smallest possible set of high-signal tokens.

## PART 2 · The 12 moves

04. Set the system prompt at the right altitude — firm rules, room to reason; labeled sections; add lines only when failures earn them.

05. Spend tokens like money — subtraction pass before important runs.

06. Retrieve just in time — don't pre-load everything.

07. Compact the conversation before it rots — /compact with keep-list.

08. Show examples instead of explaining — 2–4 clean I/O examples.

09. Label every block so the model can find it — XML or headers.

10. Put stable stuff first, key fact on the edges — top: rules/reference; bottom: current task/critical constraint; never bury the hinge fact in the middle.

11. Give the model a memory outside the window — notes file re-read after reset.

12. Hand heavy reading to a sub-agent — clean main window; short summary back.

13. Build tools like a clean API, pin the output shape — small non-overlapping tools; schemas not hope.

## Mistakes that wreck context

- Pasting everything "to be safe"
- Treating giant window as free
- Burying key instruction in the middle
- Carrying whole transcript forever
- Explaining when you could show
- One giant tangled tool list
- Memory only inside the window
- Trusting advertised context size
