# Claude Dynamic Workflows for Your Second Brain

Dynamic Workflows is the biggest Claude Code update since skills and subagents. It's not just for coders — it changes how you use your Obsidian vault.

**What it is:** Claude writes JavaScript programs that orchestrate sub-agents. Two patterns: parallel execution or pipeline (phase 1 → phase 2). Deterministic — all items get addressed, sub-agents have isolated context.

**Why it matters over raw Claude Code:**
- No agent laziness — deterministic program execution ensures all items are addressed
- No self-preferential bias — sub-agents with isolated context don't prefer their own findings
- No goal drift — isolated context windows prevent multi-turn drift

## Three Real Examples

**1. Mine last 50 sessions for recurring corrections**
Prompt: "Go through my last 50 sessions and mine them for the corrections I keep making and turn them into CLAUDE.md rules." 10 parallel Opus agents extract corrections → agent analyzes → reconcile against existing CLAUDE.md. Result: HTML report, 49 sessions analyzed, 86 corrections mined with quotes verified against actual sessions. Pattern: package recurring corrections into skills, not just CLAUDE.md.

**2. Mine daily notes for repeating patterns**
Prompt: "Go through my daily notes and extract the insights and patterns which keep repeating." Haiku sub-agent per note (31 notes, very fast) → Opus agents synthesize across all notes. Result: recurring tension points ranked, each backed by evidence with date and exact bullet point. Reproducible — same workflow next month gives comparable results.

**3. Mine NotebookLM notebook for implementable ideas**
Prompt: Extract transcripts from NotebookLM videos → analyze for ideas implementable in Obsidian vault → generate paste-ready prompts. Result: idea cards like "skill-audit workflow" with rubric scoring, paste-ready Claude Code prompt, and specific implementation steps.

## Patterns & Future Directions

- **Classify and act, fan out and synthesize** — most common Obsidian patterns
- **Tournament** — multiple attempts → judge scores → pick winner
- **Loop until done** — scheduled daily digest: get YouTube transcripts → create report
- **Deep verification** — claim extractor per claim → source checker → verified report

**Skills + Workflows integration:** Keep workflows self-contained within skills. Add a JavaScript file inside the skill that encodes the workflow. "I don't want to contaminate or blend my skills with the workflow. I just want to work with one entity, which is skills."
