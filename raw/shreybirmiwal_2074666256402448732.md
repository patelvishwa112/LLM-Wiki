## Post

In this blog, I'm going to try to explain speculative decoding in simple language by walking through the history of speculative decoding papers, because I think speculative decoding is really, really cool and isn't talked about enough. I'm going to try to share the current state of speculative decoding and what's possible next.

TLDR: Speculative decoding allows you to get up to 8x speedups on LLM workloads completely lossless and is pretty close to a free lunch in inference.

### Prereq: autoregressive LMs and hardware

Large language models generate tokens sequentially: token n depends on KV built from all prior tokens. Decode is one token per forward pass — low arithmetic intensity (FLOPs per byte loaded), memory-bound on GPUs.

Wall-clock per step ≈ max(memory time, compute time). **Arithmetic intensity** = FLOPs / bytes loaded. **Roofline**: increase intensity to move from memory-bound (point A) toward compute ridge (point B); batching too far right (point C) hurts per-user latency without more throughput.

**Batch parallel** amortizes shared weights: intensity ∝ B / (W + KV·B). Does not amortize per-sequence KV; raises latency; KV HBM limits batch size.

**Sequence parallel (prefill)** processes known prompt tokens in parallel; amortizes weights and sequence KV. Decode remains sequential because future tokens are unknown.

### Leviathan speculative decoding

Draft model guesses several tokens cheaply; target model **verifies** in one parallel forward (prefill trick on guessed suffix). Accept until first mismatch; 1..n+1 tokens per target step. Intensity ∝ (accepted tokens) / (W + KV).

Costs matter when already compute-bound: verification work scales with draft length; draft forward is additive latency.

**Lossless rejection sampling:** accept x with min(1, p(x)/q(x)); else sample residual max(p−q,0). Paths sum to p(x) — identical distribution to target-only generation.

Example vocabulary [hi, hello, hey]: p=[.5,.3,.2], q=[.2,.7,.1], draft picks "hello" → accept with .3/.7; on reject sample normalized max(p−q,0) → [.75,0,.25].

Proof sketch: propose+accept contributes min(q,p); reject+residual contributes max(p−q,0); total = p(x).

### Advances in speculators (detail)

- **Medusa (Cai '24):** heads on target last layer; ~3.5× coding; independent heads limit chain coherence.
- **Eagle 1 (Li '24):** drafter uses target hidden states; higher acceptance.
- **Eagle 2:** dynamic tree verification.
- **DFlash (Chen '26):** block diffusion drafter; ~6×; weak cross-token coupling in block.
- **SpecInfer (Miao '23):** tree attention — branch uncertain tokens.
- **DDTree (Ringel '26):** trees on DFlash; 7–8× on some workloads.
- **DSpark (DeepSeek '26):** improves DDTree (author reading list).

### What's next (author notes)

Speculation for throughput on long-context agents; disaggregated speculator hardware; MoE-aware drafters; workload-specific routing (code vs chat); online speculator training without data retention (Baseten, Modal).
