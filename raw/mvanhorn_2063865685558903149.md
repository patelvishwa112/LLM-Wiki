# WTF Is a Loop? Peter Steinberger vs. Boris Cherny

**Author:** @mvanhorn (Matt Van Horn)
**Source:** https://x.com/mvanhorn/status/2063865685558903149
**Date:** 2026-06-08

---

The most repeated sentence in AI coding this week is six words long, and almost nobody saying it can define it. One tweet had the entire timeline in a chokehold this week, so I ran /last30days on the word everyone was fighting about. The answer is real, it has a five-year lineage, and the punchline is that the loop, not the model, is now the expensive part.

## The tweet that has the timeline in a chokehold

Peter Steinberger posted it on June 7, clearing 2.2 million views: "Here's your monthly reminder that you shouldn't be prompting coding agents anymore. You should be designing loops that prompt your agents."

The most telling reply came from Varadh Jain, who asked what this looks like in practice. Matthew Berman's answer became the whole mood: "nobody knows but him and boris."

That is the real story. Not that loops are the future, but that a six-word phrase hit two million views while the people boosting it argued in the replies about what it meant.

## What a loop actually is

Boris Cherny created Claude Code as a side project in September 2024. On stage at the Acquired Unplugged event on June 2, he gave the cleanest definition:

"Now it's actually leveled up to the next wave of abstraction where I don't prompt Claude anymore. I have loops that are running. They're the ones that are prompting Claude and figuring out what to do. My job is to write loops."

A loop is a small program you write that prompts the coding agent for you, reads what it produced, decides whether it is done, and if not, prompts it again. You stop being the thing inside the loop typing prompts. You become the author of the loop. The model becomes a subroutine.

Boris's three stages:
1. Write code by hand with autocomplete
2. Run 5-10 Claude sessions in parallel, prompt each one
3. Don't prompt at all — write the loops that prompt Claude

He deleted his IDE in November and has not opened it since. In December 2025: "In the last 30 days, 100% of my contributions to Claude Code were written by Claude Code. I landed 259 PRs."

## The spectrum: from ReAct to orchestration

- **Stage 1:** Academic while-loop (ReAct, 2022) — model reasons, calls tool, reads result, repeats.
- **Stage 2:** AutoGPT (2023) — goal-driven self-prompting, infamous for spinning forever.
- **Stage 3:** Ralph loop (Geoffrey Huntley, July 2025) — bash one-liner, disciplined context reset.
- **Stage 4:** /goal command (spring 2026) — Codex and Claude Code shipped /goal running ralph loop until validator confirms done.
- **Stage 5:** Orchestration loops — the loop became the unit of work, loops supervising other loops concurrently on a schedule, with durability and crash recovery.

## It's just a cron job with a hat on

The skeptic line: "Cronjobs have funny re-branding rn."

The answer: cron runs a fixed script. A loop runs a model that looks at the current state, decides what to do next, does it, checks whether it worked, and decides whether to keep going. The decision is the agent's, not a hardcoded branch. Loops are cron plus a decision-maker in the body.

## What it looks like in practice

Starter command: `/loop babysit all my PRs. Auto-fix build issues, and when comments come in, use a worktree agent to fix them.`

Boris's five tips for autonomous Opus:
1. Use auto mode for permissions
2. Use dynamic workflows to orchestrate hundreds of agents
3. Use /goal or /loop to keep going until done
4. Use Claude Code in the cloud so you can close your laptop
5. Make sure Claude has a way to self-verify its work end to end

Tip 5 is the one practitioners obsess over: a loop is only as trustworthy as its ability to check its own work.

## The plot twist: the loop is now the expensive part

Uber capped engineers at $1,500/person/tool/month for Claude Code and Cursor after burning its annual AI budget in 4 months. The cost moved from tokens to loop management.

Production hard stops: maximum iteration count, no-progress detection, token/dollar budget ceiling.

## It's not loops. It's skills.

Steinberger's other recurring point: if you do something more than once, turn it into an automated skill. A loop with no reusable skills is a while-true around a stranger. A loop calling a library of sharp, tested, named skills is a system that compounds.

## Key Patterns

- A loop is cron plus a decision-maker in the body
- Lineage: ReAct 2022 → AutoGPT 2023 → ralph 2025 → /goal spring 2026 → orchestration loops now
- The loop is only as good as its feedback. Continuous review and validation gates are what make it trustworthy.
- The expensive resource shifted from tokens to loop management.
- The reusable unit inside the loop is a skill, not a prompt.
