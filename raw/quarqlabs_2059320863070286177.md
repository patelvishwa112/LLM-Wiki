For the past month, QuarqLabs had been focused on building an AI agent that supports actual continual learning. Most useful assistants need context that accumulates over time — they need to remember decisions, preferences, and information gathered across dozens of conversations.

Building this kind of memory is much harder than storing chat history. An agent needs to retrieve the right information at the right moment, reason across events that happened weeks apart, and avoid confidently inventing details when information is missing.

## What Makes Memory Difficult?

At first glance, memory seems like a storage problem. In practice, storage is the easiest part. The real challenges: determining which memories are relevant, which version of a fact to trust, whether multiple pieces need combining, and how to handle incomplete or contradictory evidence.

Humans navigate these ambiguities almost automatically. AI systems don't have that luxury.

## The Architecture: Four Layers

### 1. Query — Finding the Right Evidence

People rarely ask questions using the same language as stored memories. Quarq expands each question into several search hypotheses before retrieval. Each hypothesis approaches from different angles: people, actions, dates, locations, entities.

**Retrieval Pipeline:** Each query runs across both vector and keyword indexes. Vector search finds conceptually similar memories when wording differs. Keyword search provides stronger recall for exact names, dates, product names. Results are merged, deduplicated, and ranked. This combination consistently outperformed either strategy alone.

### 2. Storage — Organizing Memory by Type

Quarq separates memory into three categories:

- **Semantic memory:** Durable facts about the user — preferences, possessions, relationships, long-term attributes
- **Episodic memory:** Events that happened at a particular time — conversations, meetings, purchases, trips, decisions
- **Procedural memory:** Behavioral instructions — formatting preferences, tone guidelines, project conventions

Semantic and episodic stored as FAISS vector indices backed by JSON records. Procedural stored as lightweight JSON rule store. This separation keeps retrieval context-aware: personal history questions prioritize episodic, preference questions focus on semantic.

### 3. Reasoning — Preventing Hallucinated Recall

Even with right information available, LLMs can confuse dates, mix entity information, or confidently answer with incomplete evidence. Quarq applies explicit reasoning constraints:

- **Temporal safeguards:** Distinguish between when event happened vs. when recorded — prevents storage timestamps from becoming event dates
- **Numerical safeguards:** Determine who/what a number belongs to before using it
- **Entity isolation:** Facts about one person don't transfer to another
- **Evidence insufficiency detection:** Instead of guessing, performs another retrieval pass with more targeted queries

Guiding principle: If evidence is incomplete, search again before generating an answer.

### 4. Learning — Keeping Memory Accurate

Memory is not static. People change jobs, preferences evolve, old info becomes outdated. Quarq treats memory as maintainable over time.

After each response, a dedicated learning model reviews the interaction and determines whether to create, update, or remove memories. This runs asynchronously in background — users don't wait for memory extraction.

Over time, memory becomes more structured and accurate. Instead of accumulating duplicates, information consolidates into a cleaner representation.

## Design Principles

1. **Retrieve Broadly, Reason Narrowly** — Search aggressively, reason conservatively over evidence
2. **Evidence Beats Confidence** — Acknowledge uncertainty rather than invent details
3. **Time Is a First-Class Concern** — Dedicated temporal reasoning safeguards
4. **Learning Should Not Increase Latency** — Learning runs in background
5. **Context Is a Resource** — Large context windows create illusion of simplicity; only load relevant info

## Getting Started

```bash
git clone https://github.com/quarqlabs/quarq-agent
cd quarq-agent
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
# Configure: OPENAI_API_KEY, USER_ID, AGENT_ID
python agent.py  # terminal agent
# or: uvicorn main:app --reload  # API server
```
