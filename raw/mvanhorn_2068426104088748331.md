# WTF Is a Loop? Part 2: The 15 Loops People Are Actually Running (and the Commands to Steal Them)

**Matt Van Horn (@mvanhorn)**

This is a detailed follow-up to his previous viral piece *"WTF Is a Loop?"* (which got 3.6M views). It's about **Loop Engineering** — what loops people are *actually* running with AI coding agents like Claude Code and Codex.

### Core Distinction (the part most people get wrong)

- **`/goal <condition>`** — Runs until a verifiable outcome is achieved (e.g. "fix it until the tests pass"). Uses a separate fast model to check success after every turn. Both Claude Code and Codex support this.
- **`/loop <interval> <prompt>`** — Repeats a task on a timer *while your session is open* (hands-on monitoring). Example: `/loop 5m check the deploy`.
- **`/schedule <description>`** — Creates a cloud routine that runs while you're away (the "while I sleep" version). In Codex this maps to Automations.

There is **no `/routine` command** — the scheduler is called `/schedule` in Claude Code and Automations in Codex.

### The 15 Most Popular Loops

**From real usage across X, TikTok, Reddit, YouTube, and GitHub:**

1. **Build-test-fix pair** — One agent builds, another runs tests/lint/types and reports failures. They iterate until clean.
2. **Boris Cherny's verifier loop** — Claude Code + advanced model + independent verifier. The verifier is the part most people skip.
3. **Loop-engineer starter harness** — Ready-made template (from AI Jason) with codebase harness and knowledge base you can clone and run immediately.
4. **Five-minute repository maintainer** — Peter Steinberger's loop: every 5 minutes the agent does small verified upkeep while he's working.
5. **Plan-generate-verify-fix loop** — Bounded version with hard cap (often 5 iterations) so it doesn't run away.
6. **roborev** — Open-source tool that installs a git hook, reviews every commit, and feeds findings into an agentic fix loop. 1.4k+ stars.
7. **Goal meta-skill** — A skill that turns vague requests into rigorous goals with success criteria, boundaries, and stop conditions.
8. **15,000-emails-a-day loop** — Production email agent that classifies, drafts, and only escalates what needs human review.
9. **Anti-spin loop** — Includes explicit guards against spinning: no-progress detection, retry caps, flip-flop detection, and budget.
10. **Write-loops-not-code** — The core idea from the creator of Claude Code: instead of writing code, you write loops/routines that run overnight (30% of one person's code is now fully loop-generated).
11. **Human-in-the-loop approval queue** — Workflow pauses for human approve/revise/skip with reminders.

**Four strong ones from Matthew Berman's Forward Future Loop Library:**

12. **Production error sweep** — Reads logs, triages real errors, fixes them, and opens PRs.
13. **Quality streak loop** — Requires multiple successful realistic test runs in a row (one green test isn't enough).
14. **Adversarial-review power tool (Clodex)** — Codex reviews Claude's PRs (or vice versa) before merge. Uses `--max-iter 5` and `--threshold medium`.
15. **Completion-contract power tool** — Agent must first write a contract defining what "done" means with specific evidence before it can claim success.

### Key Warnings Repeated Across the Community

- **Cost**: Loops can burn money extremely fast. Set budgets and caps *before* you start.
- **Verification is everything**: A loop without a strong, independent verifier just automates being wrong faster. Self-grading agents will cheat (e.g. delete failing tests).

### Bottom Line from the Post

> "Stop being the thing in the loop. Write the goal, the loop, or the routine, give it a budget and a way to check itself, and go decide what to build next."

The recommended starter stack:
- Run a build-test-fix pair as a `/loop`
- Run the 5-minute maintainer as a `/loop`
- Run a PR-fixing routine as a `/schedule` overnight

All with proper budgets and verifiers.

The post ends with the practical advice that surfaced repeatedly: while your agents run, *go for a walk, call your mom, make a healthy meal, take care of yourself.*
