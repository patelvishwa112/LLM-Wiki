# Your AI's Memory Is Quietly Making It Dumber (I Cut Mine to 6 Files)

Matt Van Horn (@mvanhorn) — X Article

I gave a talk last week and said it out loud: delete that memory, agent. When your agent learns a lesson, do not park it in some private memory file. Make a pull request that improves the skill, so the fix helps you and everyone else who uses that skill.

My agent's memory had quietly grown to 218 files. A 46-kilobyte index that was so big the harness had started silently dropping the back half of it every single session. I cut it to 6 files, moved those into my CLAUDE.md, and turned auto-memory off. Then I audited the CLAUDE.md they landed in.

## Core argument

- **Push memory** (CLAUDE.md, auto-loaded memory folders) injects every session; past a point, more text means worse adherence, not more knowledge.
- **Enforced scarcity** forces distillation of what actually matters (aidecipher / TikTok framing in article).
- Keep each always-loaded file ideally **under 200 lines** (Simon Scrapes / Anthropic guidance cited).
- **39% of entries** were durable lessons tied to a single skill — belong as **PRs into that skill**, not global memory.
- **14% trash** — shipped PRs, finished work already in git history.
- Remainder: cross-cutting steering + in-flight trackers that rot when work ships.

## Hacks (article sections)

1. Bucket every memory entry: trash / skill-tied (PR into skill) / genuinely cross-cutting (keep).
2. Anything done more than twice → **skill**; clone shape from an existing good skill ("make one like Compound Engineering for X").
3. Archive full memory store outside auto-loaded dir, then delete aggressively; kept 6 survivors (safety rail, formatting/secrecy rules, canonical repo paths).
4. **Auto-memory trap:** disabling auto-memory stops writes AND reads — move survivors to `~/.claude/CLAUDE.md`, then disable (`CLAUDE_CODE_DISABLE_AUTO_MEMORY=1` or `/memory`).
5. **CLAUDE.md discipline:** delete inferable repo facts; global file can be ~19 lines; project `CLAUDE.md` as one line `@AGENTS.md` or symlink `ln -s AGENTS.md CLAUDE.md`.
6. **Pull memory** (gbrain, supermemory, mem0, Letta, Zep, Cognee) — queried on demand; complements pruning push memory; pull stores still need pruning.

## Self-audit prompt (end of article)

Grade memory store (finished-work → delete, single-skill lesson → PR into skill, cross-cutting → keep; backup first). Grade CLAUDE.md/AGENTS.md (over 200 lines, inferable content, duplicate instructions vs `@AGENTS.md` import).
