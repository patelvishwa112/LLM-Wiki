Boris Cherny from Anthropic just explained how 80% of their engineers use Claude Code every day

the part that hit me:

> give Claude access to run unit tests and it will rewrite the code until tests pass - no human intervention
> pipe massive GCP logs or git status into Claude Code via SDK with -p flag and process the output as JSON
> Ctrl+R shows you the exact context window the model currently sees
> use git worktrees to run 5 parallel Claude sessions on the same repo simultaneously
> enterprise policy files let you pre-approve safe bash commands so the model never asks for confirmation
> # before a message saves it permanently to CLAUDE.md memory for future sessions

this is the internal workflow of the people who built the model

watch it before building anything with Claude Code
