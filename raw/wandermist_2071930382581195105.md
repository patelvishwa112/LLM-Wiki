# I Made My Hermes Agent 10x Faster Without Changing the Model

**Posted:** Jun 30, 2026

Thesis: agent **capability** is often wasted on **navigation** when vaults are organized for human content-types, not agent retrieval paths.

## Failure mode

Content-type folders (articles, research, assets) scatter one task across many locations. Agent searches broadly, opens wrong/archived files, asks disambiguation questions—across Opus, GPT, Qwen, GLM.

**Before reorg (directional timings):**
- Current article brief: 7 files, ~2:00
- Brand colors: 5 files, ~1:12
- Article queue: 4 files, ~0:48
- Same-topic articles: 6 files, ~1:36
- Launch promotion strategy: 3 files, ~0:34

## Fix: concern-based folders + INDEX.md

Three rules:

1. **INDEX.md at root of each major folder** — subfolder map, canonical files, "Where To Go" starting points (not exhaustive file lists; ~third version sweet spot).
2. **One folder per concern** (brand system, editorial, promotion, etc.) with numbered subfolders for read order (`01.`, `02.`, …).
3. **06.Archived/** — inactive material; agent skips unless historical context requested.

Example brand tree:
```
05.Brand/
├── INDEX.md
├── 01.Brand System/
├── 02.Editorial Strategy/
├── 03.Promotion/
├── 04.Public Deliverables/
├── 05.Operating Plan/
└── 06.Archived/
```

**After reorg:** slowest task ~26s (was ~2:00); many tasks ~10–18s with 1–2 file opens.

## Pitfalls called out

- INDEX in every subfolder → index-reading overhead; only major folder roots.
- Over-nesting (5 levels) → multiple INDEX passes; prefer flatter.
- Structure before demonstrated confusion → add minimal scaffolding when agent actually gets lost.
- Sync lag (e.g. Obsidian across devices) can surface stale files—separate from structure but looks like navigation failure.
- Numbering past ~10–12 top-level items loses meaning.

## 5-minute diagnostic

Time 3 frequent tasks: >30s or ≥3 wrong opens → write INDEX.md for that folder's root; rerun. Iterate one folder at a time.

## Closing frame

"Capability is cheap when scaffolding is broken" — same theme as owning provider/tool layer; folder INDEX as smallest high-leverage harness piece for file-heavy agents (Hermes cited).
