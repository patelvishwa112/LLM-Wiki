## Post

In most conversations about agentic engineering, the action has changed from prompting to operating. Software factories, goals, loops, background sessions, subagents, hooks, sandboxes, agent-approving agents. Claude Code and Codex expose the shift directly.

From the engineer standpoint: low autonomy to limit risk and increase reversibility; higher autonomy for explicit activities; fleets of parallel agents on massive codebases. Core question: what level does this task deserve, and what verification makes that level defensible?

Manager agent on triggers: delegate, verify continuously, return only human decisions. Steve Yegge single-axis ladder (Gas Town / Pragmatic Engineer) is a proxy for single-agent trust but cannot place multi-agent skill.

Separate two questions: how far a single agent goes (agency) vs skill coordinating many agents (orchestration).

**Agency axis:** low = suggest and wait; mid = scoped task with evidence and steering; high = goal-seeking with experiments, blocks, questions, evidence.

**Orchestration axis:** low = one agent one thread; mid = several agents in isolated worktrees; high = orchestrator + queue, management by exception.

Claude Code: /plan, /goal, /loop, /background, /batch, /code-review, hooks, subagents, checkpoints, agent teams, /schedule. Codex: Goal mode, worktrees, Automations, subagents, Auto-review, sandboxing, etc.

**Three eras (six levels):** assist → supervised → scoped delegation → goal autonomy → parallel delegation → managed-by-exception factory.

**Levels 0–5** (Assist, Supervised action, Scoped task delegation, Goal-driven autonomy, Parallel delegation, Managed-by-exception orchestration) — each with use cases, verification shift, failure modes (approval fatigue, false parallelism, merge conflicts, orchestration tax).

**Risk:** How fast do we know we're wrong? How cleanly undo? What proves right? If answers are slow, hard, and "trust the summary" — not high autonomy.

**Agent contract:** goal, scope, non-goals, tools/permissions, stopping condition, evidence, escalation, budget (time/tokens/retries/parallelism).

**Metrics:** MTBI, longest unattended run, sandbox share, auto-approve rate, actions per human instruction, clarification/interrupt rates, review time, rework, defect escape, token cost per accepted change.

**Anti-patterns:** autonomy as status, permission laundering, summary substitution, fleet cosplay.

**Climb safely:** one axis at a time; verification follows risk/reversibility not task name.

Closing: calibrated autonomy; verification remains the bottleneck.
