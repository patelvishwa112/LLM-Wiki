# How we built secure automated learning loops in Modal and Claude Code

**Olivier Morissette** (@oliviersm199) — CTO at Terse (useterse.ai); former tech lead at Squarespace and New York Times. With Thomas Karatzas.

The hype the last couple of weeks has been around implementing loops and to quote Satya Nadella: *AI systems that improve with each use.* The last few months, Thomas Karatzas and I have been heads down building Terse, an AI workflow platform. This article shares how we built automated learning loops.

## What the feature is

Terse runs an automated review of every workflow that gets deployed. Once a week it drops in your inbox with concrete suggestions for how to improve your workflows.

We built this because software is rarely perfect the first time. Edge cases and real-world examples only get worked out through initial use. Reviewing workflows is time consuming and involves sifting through piles of logs. AI reduces this friction and helps unlock value faster.

Once you receive the proposed improvements, you can review the advice, then use the **terse apply** CLI to apply any improvement to your workspace automatically.

## How it works on the backend

Four-step process:

1. Take the tenant's deployed workflow source (from their last `terse deploy`) and production logs as context.
2. Spin up an isolated **Modal sandbox**.
3. Run the **Claude Code CLI** and provide the workflow context.
4. Return structured patch suggestions, which the user reviews with `terse list improvements` and applies with `terse apply <improvement-id>`.

## Threat model

The inputs to Claude Code put attacker-controlled data straight into an agent with tools. A malicious user could upload source code containing a prompt-injection payload, or plant one in run-history logs. Initial implementation had five issues:

1. LLM calls made directly with a raw Anthropic API key.
2. Open egress to the entire internet.
3. The key had no expiry, no spending limit, and no scope.
4. Every Claude Code tool was available.
5. Nothing in the prompt told the model which input was untrusted.

Worst case: hijacked Claude Code sends sensitive information to an attacker's server or runs up a massive Anthropic platform bill.

## How Modal provides a security foundation

Modal Sandboxes run untrusted code; infrastructure attack surface was handled:

- **gVisor** user-space kernel intercepts syscalls instead of handing them to the host.
- Fresh, per-execution filesystem destroyed on teardown.
- Resource and timeout limits enforced per run.

## Application-layer defense in depth

### Layer 1: Egress lockdown to the proxy only

Restrict sandbox egress to a specific IP range at creation — pinned to a **LiteLLM** proxy for model calls.

```typescript
const PROXY_EGRESS_CIDRS = ["<litellm-proxy-cidr>"];
// ...
egressCidrAllowlist: PROXY_EGRESS_CIDRS;
```

### Layer 2: Prebaked image

Dependencies (git, Claude Code) prebaked into the image; sandbox cannot download at startup due to Layer 1.

### Layer 3: Ephemeral and scoped tokens

LiteLLM virtual keys (compared to Visa debit cards):

- Short-lived, expire automatically
- Budget-capped per run
- Rate-limited (RPM cap)
- Revoked immediately after the run
- Compatible with Claude Code

Anthropic workspace spend limits lack per-run narrowly scoped keys — hence the proxy.

### Layer 4: No credential on disk

LiteLLM virtual key passed via Modal API into process environment; Claude Code runs as non-root `coder` user with escaped arguments; tokens not written to disk.

### Layer 5: Claude Code tool denylist

Written to `/tmp/project/.claude/settings.json` after unzip — denies Bash(curl/wget/nc/ssh/...), WebFetch, WebSearch, etc. Defense in depth on top of egress lockdown.

### Layer 6: Prompt sanitization

`promptSanitize` utility for every tenant-controlled field:

- Wrap in `<untrusted field="...">...</untrusted>` tags
- Strip control characters
- Truncate to per-field max length
- Rewrite embedded `</untrusted` or `<untrusted` to `<_untrusted_`
- Preamble instructs model to treat tagged content as raw data only, ignoring instructions inside

## Takeaway

Modal provided trusted infrastructure isolation so effort could focus on application threats. Secure automated improvement loops make agent refinement friction near zero as real usage feeds back into workflows.
