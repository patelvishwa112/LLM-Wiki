# 21 Painful Mistakes I Made Building AI Agents

By Graeme (@gkisokay) | May 28, 2026 | 11K views | 282 bookmarks

3 months of building Hermes and OpenClaw agents every day.

1. Don't build one giant agent — crew of specialized agents with clear ownership
2. Don't build output-first — research agent first as input intelligence layer
3. Don't confuse scraping with research — structured, verified, source-backed
4. Don't let research die in a doc — route findings to coding, content, marketing
5. Don't let autonomous workflows run blind — supervisor monitors intended vs actual flow
6. Don't run OpenClaw solo — Hermes as supervisor with persistent memory
7. Don't auto-build before auto-think — self-thinking layer first
8. Don't give agents vague goals — define "done," acceptance criteria, recovery logic
9. Don't build from loose plans — clarify requirements before implementing
10. Don't accept output without proof — test, verify, cite, demonstrate
11. Don't scale without cost tracking — log exact cost per run
12. Don't use frontier models for everything — local/cheap for scanning, frontier for reasoning
13. Don't depend on one model/provider — mix of GPT-5.5, Minimax, local Qwen
14. Don't ignore local LLMs — always-on layer for 24/7 background cognition
15. Don't let agents go stale — weekly audits after tool/model/MCP updates
16. Don't build content agents with only voice — taste, thesis, proof, forbidden-pattern files
17. Don't think the model is the product — system around it: research, routing, memory, supervision
18. Don't use expensive tools — Grok for research, $30/mo instead of X API
19. Don't chase AGI before reliability — boring infrastructure first
20. Don't use someone else's setup — learn from building your own
21. Don't be afraid to share — build in public
