## Post

> LLMs compressed the internet into weights. Agents need to compress work into state.

## The Core Thesis

LLMs are among the most successful artificial memory systems ever built — they compressed public text and code into weights via next-token prediction. Agents need a second kind of memory, built around consequence rather than prediction: decisions, corrections, commitments, exceptions, dependencies, handoffs, failed attempts, and workflows that exist in people's heads, not systems of record.

"Knowledge is what was present. Memory is what becomes useful."

- Knowledge = what was present: the document, meeting, customer complaint, code diff, ticket
- Context = the situation that turns stored material into something usable: who is asking, what they're trying to do, risk, what action follows
- Memory = the subset of the past that should survive because it changes future behavior

"The shortest version: memory is retained consequence. Purpose is the utility function that decides which parts of the past deserve to survive."

## The Boulder Problem

A boulder beside a hiking trail has size, shape, material, location, texture, moss, slope. Those are facts about what is present. But:
- For a tired hiker, it's a seat
- For a navigator, it's a landmark
- For trail maintenance, it's an obstacle
- For a geologist, it's a sample
- For a poet, it's an image
- Someone else forgets it before the next bend

The boulder didn't change. The utility function did. Meaning appears when knowledge is evaluated under a purpose.

## The Company Version

A customer email is not one memory:
- Sales: renewal signal
- Product: missing feature
- Engineering: technical constraint
- Legal: obligation
- Customer success: escalation context
- CEO: evidence of a market pattern

If the system stores it as a complaint, it helps support and distorts product. As a feature request, helps product and misses legal. As churn risk, helps revenue and causes engineering to ignore the architectural issue. Each label is valid inside one frame, misleading inside another.

"A customer complaint becomes a support thread, then a Slack debate, then a product decision, then an engineering ticket, then a customer promise, then a renewal risk. The memory is not any one artifact. The memory is the state change across the chain."

## Memory as Highest-Value Compression

Memory compresses the past into future-relevant state:
- The fact that should wake up later
- The correction that should change the next run
- The promise that should constrain the next email
- The failed workaround that should not be retried
- The objection that appeared across ten calls
- The architectural invariant hidden in an old comment

Most surrounding noise should disappear. A system that remembers everything has postponed judgment. The work is knowing what can vanish, what must remain exact, what should become a rule, what should decay, what should stay attached to evidence.

"Forgetting is not the opposite of memory. Ungoverned forgetting is a failure, but governed forgetting is part of intelligence."

## The Price of Meaning

Semantic retrieval works because related concepts are placed near each other — but the same proximity that lets a system generalize also creates interference. Similar memories become neighbors, distinctions get compressed. The system retrieves something plausible because it is close, not because it's the right state.

In "Geometry of Forgetting" and "The Price of Meaning," Ashwin argues that semantic memory lives on a frontier between usefulness and interference. Semantic memory needs exact episodic grounding as counterweight: source material, time, evidence, provenance, permissions, contradictions, corrections, outcomes.

## Bigger Context Is Not Memory

Bigger context windows solve the wrong problem: they give the model more to read when the real goal is to know what should no longer require rereading. Cheaper tokens reduce the pain of waste but make sloppy context feel affordable.

"The cheapest token is the one never spent because the system already knew what mattered."

## Graphs Are Projections

A graph is a commitment: these are the nodes, these are the edges, these labels are worth preserving. That commitment is useful once the task is known. Before the future question exists, the same commitment can trap the system inside a frame that is clean, structured, and wrong for the work.

"The danger is not that the graph is wrong. The danger is that it is prematurely right."

**Semantics at ingestion. Ontology at retrieval.** At ingestion, preserve the richest possible semantic substrate: actors, artifacts, actions, properties, timestamps, modality, source, uncertainty, evidence, permissions. At retrieval, let the task supply the ontology. The actor, role, question, permission boundary, and risk profile determine which relationships matter.

## What a Real Memory Substrate Requires

Five things:
1. **Exact episodic record** — source material: traces, meetings, messages, documents, tickets, code changes, approvals, corrections, outcomes
2. **Shared semantic state** — actors, artifacts, actions, properties, timestamps, sources, evidence, uncertainty, relationships, permissions
3. **Purpose layer** — tells the system which parts of the past should matter now: actor, role, task, risk, goal, time horizon, permission boundary, expected action
4. **Governed consolidation and forgetting** — some memories remain exact, some summarized, some become rules, some decay, some archived
5. **Action feedback** — when an agent acts, a human corrects, a workflow succeeds, a customer renews, a bug reappears, the result feeds back

## Company Brain (Sentra)

Three views over one shared state:
- **Factual memory:** what exists and what happened — docs, tickets, customers, calls, code, dashboards, incidents, owners, artifacts
- **Interaction memory:** what people and agents meant, debated, promised, implied, contested, left open — across emails, messages, meetings, comments
- **Action memory:** how work actually gets done — the lived workflow, the trigger, the guardrail, the outcome that should alter the next run

"These are not three products. They are three views over one shared state, which is why splitting them apart too early recreates the same memory problem inside the memory product."

"The model generates the answer. Memory decides the world."

## The Memory Market

The market is circling the same missing layer from different directions: Mem0 (personalization), Zep/Graphiti (temporal truth), Letta (runtime state), Hindsight (epistemic structure), Glean (enterprise search), Google Agentspace, Zapier, ServiceNow, Granola, Otter. Each wedge is right about something. None answers the Company Brain question alone.

## Closing

"LLMs proved that compression can produce intelligence. They compressed public language into weights under next-token prediction. The next compression problem is private, changing, permissioned, and action-bearing: what should survive from the past because it changes what happens next."

"If agents become the way companies operate, the memory layer has a credible path to becoming the control plane for applied intelligence. The model generates the answer. Memory decides what world the answer belongs to."
