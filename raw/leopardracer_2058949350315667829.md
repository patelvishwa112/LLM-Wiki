If you're a finance person, you've probably looked at Claude Code once and looked away. A black terminal. Commands that mean nothing. Something for engineers.

I've now been running my entire investment research workflow on Claude Code for sixty days. Screening, scraping, modeling, drafting reports, monitoring portfolios. The terminal stopped being intimidating somewhere around day three.

Here's what nobody told me: Claude Code is for coding if you want to code. It's for cooking if you want to cook. It's for shopping if you want to shop. And if you're really into finance, Claude Code is for finance too. The "code" in the name is the most misleading part of the product.

## Claude vs. Claude Code: The One Thing Nobody Explains

Claude and ChatGPT are chatbots. You ask, they answer. Claude Code is Claude with hands.

A chatbot can help with one step at a time. Claude Code can run the whole chain. You give it the end goal. It opens a browser if it needs to. It scrapes data if it needs to. It writes files. It reads files. It runs five things in parallel if the task allows. It asks you for input only when it actually needs you. The work happens while you do something else.

A smart, hardworking person does more by doing more. A wise person does more by eliminating more. Claude Code is the eliminator.

## Step One: Install Claude Code

Go to the Claude Code page on Anthropic's site. Copy the install command for your machine. Paste, hit enter, wait a minute. Type `claude` and you're in. Also download Cursor (cursor.com/home) — it's where the Obsidian knowledge base, chat, and tools are all visible at the same time.

## Step Two: Set Up Obsidian as Claude Code's Memory

Claude Code on its own has no memory. Every session starts fresh. Obsidian solves this — it's a free app that turns a folder into a personal research library. Each note is a plain text file. Claude Code can read and write to that same folder.

### Folder Structure
- `raw/` — inbox where you dump source material
- `wiki/` — Claude Code's domain, structured summaries and master index
- `output/` — where finished work lands

### CLAUDE.md (The Rulebook)
The most important file in the system. Claude Code reads it every session. It tells Claude Code:
- It is the librarian, wiki/ is its domain
- raw/ is the inbox, process during "compile"
- `wiki/_master-index.md` is the entry point
- Each topic gets its own subfolder with `_index.md`
- Always use `[[wiki links]]` to connect concepts
- Compile workflow: read raw → identify topic → write wiki article → update indexes
- When answering questions, navigate master-index → topic index → articles

### Daily Workflow: Four Verbs
1. **Clip** — One-click browser extension saves articles to raw/
2. **Compile** — Weekly: Claude Code processes raw/ into wiki/
3. **Query** — Ask the library questions, get cross-referenced answers
4. **Audit** — Monthly: Claude Code reviews wiki for inconsistencies

### Web Clipper
Install Obsidian Web Clipper extension. Set template: vault = your vault, note location = raw, note name = `{{date|date:"YYYY-MM-DD"}}-{{title|safe_name}}`

## Step Three: Give Claude Code Hands on the Web

### Playwright
Lets Claude Code drive a real browser — navigate pages, click buttons, fill forms, log into accounts. For finance: screeners that need filter setup, IR pages with hidden data behind dropdowns.

Install: ask Claude Code to walk you through Playwright CLI installation from github.com/microsoft/playwright-cli

### Firecrawl
Fast programmatic web scraping. Pulls clean markdown from any page. Use cases: pull insider trading from Finviz, scrape annual reports, search for news and pull top articles.

Install: ask Claude Code to walk through Firecrawl CLI from docs.firecrawl.dev/sdks/cli

### Real Example
"I'm screening for names to deploy capital. Go to finviz.com, pull insider trading data for the last two weeks of US-listed companies. Analyze for strong buy signals. Give me a top 10 watchlist with reasoning."

Claude Code used Firecrawl, parsed data, ran buy-signal logic, cross-checked fundamentals, returned ranked list of 10 names with thesis, insider purchases, risks, and scorecard. Three minutes start to finish. Manual would have been 90+ minutes.

### Permissions
The `--dangerously-skip-permissions` flag turns off tool approval prompts. Author ran with it for 60 days without issues. CLAUDE.md provides the guardrails since Claude Code only has access to the vault folder.

## What 60 Days Has Taught Me

1. **Terminal intimidation is temporary** — gone by day three
2. **From chatbot user to workflow operator** — stop asking for help with one task, start handing off the whole task
3. **Research used to leak** — scattered across Notion, Apple Notes, bookmarks, PDFs. Now everything lands in one place, gets filed, and compounds
4. **NotebookLM** — Google's hosted tool for focused research on a single company/sector. Python wrapper lets Claude Code query it directly
