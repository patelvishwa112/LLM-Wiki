# Prompt Caching with Deep Agents

Alex Olsen (@its_ao) — LangChain / Deep Agents

**Claim:** Prompt caching can cut inference token cost **41–80%** for agents at scale. Manus AI quote: KV-cache hit rate may be the single most important production agent metric.

**Problem:** Provider caching strategies differ (explicit breakpoints, implicit longest-prefix, TTL, prewarm, routing keys) — hard to maximize savings when switching models.

## TL;DR — how prompt caching works

Each turn reprocesses system prompt, tools, skills, history, new message. Caching stores model state after a prefix; next request resumes from snapshot. New skills/tools can **bust** cache by changing earlier prompt text. Explicit breakpoints (where supported) limit bust radius.

**Provider feature matrix (article, check docs for updates):**
- Explicit breakpoints: Anthropic ✅, Gemini ✅; OpenAI ❌ (implicit longest-prefix); Fireworks/Baseten vary
- Configurable TTL, cache prewarm, routing key — uneven support

## Deep Agents approach

Automatic best-effort:
1. Explicit cache breakpoints when supported
2. Implicit provider caching otherwise
3. Prompt structure to maximize cache reads

Provider detected → provider-specific middleware. `createDeepAgent({ model })` — caching on by default. LangChain `createAgent` + `anthropicPromptCachingMiddleware()` for opt-in.

**Blast-radius control:** Static prefix (tools, skills, system) should stay stable; memory updates / compaction can bust — Deep Agents places cache points so partial prefix still cache-hits.

## Eval savings (Deep Agents suite, mid-tier models)

| Model | Token cost reduction |
|-------|---------------------|
| claude-haiku-4-5 | -77% (explicit breakpoints) |
| gpt-5.4-mini | -80% (automatic longest-prefix) |
| gemini-3.5-flash | -49% (implicit, no guarantee) |

Longer trajectories benefit more (prefix reused every turn).

## LangSmith observability

Per invocation: TTFT, input tokens, **cache-read tokens**, output tokens; trajectory rollups. Used to produce article numbers (eval → dashboard → SDK → notebook). Separates caching savings from trajectory length / cheaper turns.

## Future

Prewarm, routing keys, configurable TTL as providers converge. `createDeepAgent` to absorb new features without extra config.
