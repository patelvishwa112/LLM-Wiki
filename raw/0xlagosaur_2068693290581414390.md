# How to Build a Claude Skill (So You Never Paste the Same Prompt Twice)

**@0xLagosaur** — X Article, Jun 2026. Full text via Playwright.

---

## Core shift

- **Prompt** = you remember to use it every time.
- **Skill** = Claude loads it when the task matches your trigger description.

Structure: folder + `SKILL.md` with YAML header (`name`, `description`) + plain-English instructions.

## Three facts

1. Same skill folder works in Claude app, Claude Code, API.
2. **Lazy load**: only description in context until triggered (~20 skills, low overhead).
3. Open standard (late 2025): Copilot, Cursor, Codex, 40+ tools. Official examples: github.com/anthropics/skills

## Five-minute build

1. Folder `tighten-draft/`
2. `SKILL.md` with `description: Use when I ask to edit, tighten...` + editor rules
3. Upload: Settings → Capabilities → Skills

**Trigger rule:** "Use when I…" + exact phrases you type. Vague = never fires.

## Four starter skills (full SKILL.md in article)

1. **my-voice** — voice rules, banned words, formatting
2. **fact-check** — Verified / Unverified / Wrong per claim
3. **should-i** — ship / fix / kill decision filter
4. **checkpoint** — compress long chat <200 words handoff

## Pro moves

- **references/** for brand guides, examples (layer 3 lazy load)
- Layers: description (always) → instructions (on trigger) → reference files (on need)
- Debug: never fires → broaden "Use when I" phrases; fires too often → narrow; ignores rules → one job per skill, shorten
- Rule: pasted 3+ times or explained preference 2+ times → make it a Skill

## Closing

Best leverage = encode best thinking once; every session starts calibrated.
