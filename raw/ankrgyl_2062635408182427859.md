# How we made continuous trace intelligence possible at scale (Braintrust Topics)

If you run an agent in production, some part of your day is probably going through countless production logs to see if anything looks interesting. Braintrust's Topics is the solution.

**The problem:** Agent traces don't look like anything standard tools expect. A single trace can be millions of tokens — conversation history, tool calls, intermediate reasoning, retrieved context. If you embed the raw trace, you get noisy clusters dominated by surface features. If you summarize with an LLM, you blow your budget. If you sample aggressively, you miss the long tail where bugs live.

**The insight (inspired by Anthropic's Clio paper):** Instead of embedding the raw trace, ask an LLM to summarize the trace along a single dimension in a sentence or two. Then embed THAT summary, cluster the embeddings, and name the clusters with a second LLM pass. The trace itself never has to fit in an embedding model's context window.

## The 6-Stage Pipeline

1. **Preprocess:** Walk every span, parse as LLM conversation, deduplicate messages, drop scorer spans. Hard-capped at 128K tokens before reaching the facet model.

2. **Facet:** LLM produces a short text blob per dimension (Task, Sentiment, Issues, or custom). Key optimization: facets are batched into a single LLM call — trace tokens paid once regardless of how many facets you run.

3. **Embed:** Embed the facet output, not the raw trace. A consistent, short, on-topic summary embeds cleanly. A million-token agent trace does not.

4. **Cluster:** HDBSCAN with UMAP on up to 50,000 facets. HDBSCAN doesn't require pre-specifying cluster count and naturally identifies outliers as noise. Keyword extraction via c-TF-IDF (BERTopic approach).

5. **Name:** LLM produces short name + description for each cluster. Cluster identity is stable across regenerations; names may vary slightly. Similar clusters automatically match to predecessors.

6. **Classify:** No LLM call. Just nearest centroid lookup against saved topic map. ~100ms per trace. Returns no_match instead of forcing bad labels.

## Operational Properties

- **Continuous, not batch.** Runs in background: accumulate → recompute topic maps → backfill recent traces → idle until next run.
- **Minimum scale:** 400 traces per project, 100 unique facet summaries before generating a topic map. Below that, clusters aren't meaningful.
- **Two views:** Automated topic map (persistent, for trend dashboards and alerting) and ad hoc clustering (exploratory, on any subset).
- **All inference runs on Baseten.** Facet model sees only preprocessed text (no attachments/metrics). Embedding model sees only facet output. US/EU hosting, HIPAA compliant.
- **Customizable:** Drop in a custom JavaScript preprocessor for unusual trace shapes.
