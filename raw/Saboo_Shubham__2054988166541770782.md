## Content

The ultimate guide to /goal

/goal is not a feature. It is a primitive. HTTP is a primitive. JSON is a primitive. /goal is becoming one for coding agents.

OpenAI's Codex CLI added /goal. Claude Code added it. Hermes Agent has had /goal built in. So I now have a builder, a reviewer, and an orchestrator that all accept the same instruction format.

What /goal actually is: A regular prompt asks an agent for the next response. /goal flips that. You write down what 'done' looks like, submit it once, and the agent works toward it until it gets there.

The shift is from prompting (you driving) to assigning (the agent driving toward a target you defined).

The three tools that currently speak /goal: Codex (OpenAI's coding CLI, strong at implementation), Claude Code (Anthropic's coding CLI, strong at review), Hermes Agent (orchestrator that coordinates work between coding workers).

What matters isn't that any one of them shipped /goal. It's that three different teams converged on the same primitive, and that convergence is what makes it possible to compose them.
