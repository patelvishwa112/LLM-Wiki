LAYER 1: The Signal Detection Engine

Renaissance doesn't trade on news. It doesn't read 10-Ks. Simons' team relies on data-driven methods, using statistical analysis and computational power to identify market inefficiencies - collecting enormous amounts of data and analyzing it to find statistical patterns and non-random events across a wide range of markets.

The first layer is a pattern extractor. Think of it as a seismograph - not reading earthquakes, but reading the invisible tremors in price data that precede large moves.

Python skeleton:
- Autocorrelation: is today's return predictive of tomorrow's? (acf_1)
- Z-score deviation from rolling mean
- Hurst exponent: <0.5 = mean-reverting (fades work), >0.5 = trending (momentum works)
- Regime detection: switches strategies based on hurst

LAYER 2: The Factor Model (Why Stocks Move Before Anyone Knows Why)

Renaissance identifies why price moves happen by decomposing returns into factors. Fama-French 5-factor is the public version. Renaissance's private version goes 200+ factors deep.

Data sources: options chains, order-book depth, fundamentals (earnings, revenue, cash flow), corporate actions, news sentiment, social media volume, alternative data (satellite imagery, credit card transactions, web traffic, app downloads, shipping containers).

Factor decomposition via linear regression: High alpha + low R-squared = edge not explained by known factors. That's Medallion territory.

LAYER 3: The Markov Chain Regime Machine

Markov Chains model the probability of transitioning between market states: Bull (0), Bear (1), Stagnant (2).

Transition matrix example:
- From Bull: 78% stay Bull, 14% go Bear, 8% Stagnant
- From Bear: 20% recover, 68% stay Bear, 12% Stagnant
- From Stagnant: 25% Bull, 23% Bear, 52% stay

Core function: given current state, compute probability distribution over states after N steps. If 76% probability of remaining in Bear over next 5 periods — you're not going long.

LAYER 4: The Neural Network Edge Extractor

Neural net extracts non-linear relationships no formula can capture. Architecture: Linear(50→128) → ReLU → Dropout(0.2) → Linear(128→64) → ReLU → Linear(64→3) → Softmax. Outputs: [long, short, flat] probabilities.

Feature vector combines: last 20 returns, factor exposures, regime probabilities, hurst exponent, z-score.

Key insight: you're not predicting price. You're predicting probability of direction. Even 52% accuracy — if Kelly-sized correctly — prints money.

LAYER 5: The Execution Layer (Where 80% of Quants Bleed Out)

Renaissance must have built infrastructure to keep execution costs very low — the reported gross returns are after trading costs, making Medallion's performance even more extraordinary.

Kelly position sizing: half-Kelly for safety, hard cap at 2% max risk per trade. Execution cost calculation: spread + market impact. Only take trade if edge > 1.5x execution cost.
