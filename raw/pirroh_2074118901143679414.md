# Continual Learning for AI Agents — Michele Catasta (@pirroh)

Posted: 2026-07-06

Everyone talks about Continual Learning as if it means one thing only: updating model weights. But there's an inconvenient truth about the agent ecosystem — the vast majority of agents in production today leverage closed frontier models. When you don't own the weights, you certainly can't fine-tune them. For most agent builders, weight-level continual learning is off the table, especially when working at the very frontier of capabilities (think Fable 5 or GPT 5.6).

That doesn't mean agents can't learn. Agentic systems can improve at three layers — model, harness, and context — and the last two are fully within your control. Harness-level learning mines production traces to improve code, tools, and instructions; context-level learning personalizes at agent, user, and org level. Compounding improvements can ship daily.

## Evaluating and improving Replit Agent at scale

Replit Agent users describe goals in natural language without a repo, test suite, or framework — vibe coding. Success: the app works when users click around. Evaluation must be part of the improvement loop, not a one-shot launch gate.

**Three pillars:** offline benchmarks (regressions pre-ship), online A/B tests (real behavior), Telescope (trace clustering). Swiss-cheese model — no single layer suffices.

SWE-bench and Terminal-Bench miss vibe-coding signal: agents create codebases from scratch; functional correctness is whether the artifact matches the PRD.

### ViBench

Public benchmark for vibe coding from anonymized production PRDs. Agent builds from scratch; eval agent + Playwright explores invented UI/structure using natural-language test plans. Variants: Vibe-to-ref, Vibe-on-Vibe on existing codebases. Lessons: frontier coding-benchmark scores do not always transfer; models often worsen when extending their own code.

### A/B testing

Most agent-affecting updates are A/B tested. Interpretation is hard (duration, cost, sentiment ambiguous).

### Telescope

Trace clustering: facets, embeddings, density-based clustering (Clio-inspired). Turns failures into actionable clusters.

### Self-improvement loop

Reads clusters/failures → hypotheses → draft PRs with evidence → ViBench + A/B → ship/iterate/drop. Engineers own direction, eval design, architecture, launch approval.

**Example:** Cold-start environment setup degradation caught via Telescope cluster; patch + regression test; shipped same day.
