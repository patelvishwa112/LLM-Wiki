## Oak Lab blog (linked)

Source: https://oaklab.ai/posts/learning-from-experience-instead-of-curated-datasets.html
Date: July 13, 2026

Learning from experience is different from learning from curated datasets. Algorithms that learn from curated datasets can assume that all data is useful for learning. This is usually guaranteed by humans, who collect, clean, and filter raw data. Experience does not always contain learnable associations.

Demo progression (supervised prediction):
1. Single binary feature (1% ones) equals target — linear weight of 1 solves it with SGD MSE steps.
2. Add rare ±1 target noise (1%) — only targets when feature=1 remain predictable.
3. Add Gaussian target noise (var 5) — predictable component hard to see by inspection.
4. Add 4095 more Bernoulli features uncorrelated with target (4096 total) — SGD on linear model absorbs noise into weights rather than learning only the predictable component. SGD/Adam/RMSProp credit all non-zero-gradient params for all errors; no mechanism to skip unpredictable targets or useless signals.
5. IDBD (Sutton 1992) on same linear problem learns only the predictable aspect; can assign different credit per signal and withhold credit when target is unpredictable.

Non-linear: NetworkIDBD on NoisyMNIST — MNIST digit embedded in 64x64 with noisy Bernoulli border; target ±1 for odd/even digit when present (10% of steps), 0 otherwise, plus Gaussian noise. Visualize large weights: SGD increases outgoing weights from all inputs (forwards noise); NetworkIDBD concentrates on the center 28x28 digit region.

Thesis: SGD-family minimization of all errors is a fundamental reason systems fail on unfiltered online streams. Mini-batches / replay side-step by averaging noise but poorly fit online continual learning. Bet: methods that *learn* credit assignment (e.g. NetworkIDBD) enable learning from experience.
