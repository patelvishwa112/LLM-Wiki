# PorTAL: Portable Task Adapters for LLMs

## Abstract

Parameter-efficient fine-tuning (LoRA) adapts a frozen LLM but locks the adapter to one base model. **Portable task adaptation** learns a base-agnostic task latent `z_t` and hypernetwork decoder `D_b` that generates per-layer LoRA adapters. Decoder = shared core + thin per-base converter. To port: freeze `z_t` and shared core; refit only converter on small calibration data.

**Results (headline):** Task latent + core trained on Qwen3-1.7B and 4B → refit converter on unseen Qwen3-8B recovers **~98%** of per-task LoRA lift vs Cross-LoRA **~14%**. Cross-family Gemma-3-4B **~94%**. ~half the calibration data to match from-scratch LoRA plateau; better held-out log-loss calibration.

## Motivation

Foundation model release cadence accelerates; per-(task, model) LoRA retrains don't amortize. Goal: pay task adaptation once, port with light per-base refit (Platonic Representation Hypothesis inspiration).

## Method sketch

- Task latent `z_t` (d_z=256), FiLM-conditioned hypernetwork trunk, per-module heads → core-width LoRA factors, aligner `P_in`/`P_out` to base dimensions.
- Train `{z_t}`, `D_b` with frozen base; gold-continuation NLL on answer tokens.
- Multi-base training: gradient-norm balancing on `z_t` across bases.
- **Porting:** freeze core + latents; optimize only `{e_ℓ, P_in, P_out}` on calibration set.

## Experiments

- **14 MC tasks:** TruthfulQA, SuperGLUE subset, BoolQ, ARC, HellaSwag, OBQA, WinoGrande, CSQA, SciQ, etc.
- **Seen:** Qwen3-1.7B, 4B. **Unseen:** Qwen3-8B, Gemma-3-4B.
- Metric: length-normalized acc_norm; recovered lift = (acc_m - acc_base)/(acc_LoRA - acc_base).

### 6.1 Source (4B)

Base 0.627; per-task LoRA 0.765; LoRA Hypernet 0.757 (~94% lift recovery).

### 6.2 Within-family (8B)

Per-task LoRA 0.795; Cross-LoRA 0.685 (~14% lift); joint train z+D on 8B 0.785; **PorTAL port 0.792 (~98%)**.

### 6.3 Cross-family (Gemma-3-4B)

Per-task LoRA 0.778 from base 0.595; PorTAL 0.767 (~94%).

### 6.4 Data efficiency

PorTAL matches per-task LoRA peak with ~half the data; lower log-loss at all sizes on 8B sweep.

## Future work

Hard tasks underfit in shared rank-8 decoder (OpenBookQA, WinoGrande, HellaSwag); text-description encoder variant (Text-to-LoRA style); larger generative tasks.
