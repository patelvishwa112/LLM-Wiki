# RLM Agents live healthier when they talk via Structured Outputs

**Author:** @neural_avb (AVB)
**Source:** https://x.com/neural_avb/status/2063907440509571354
**Date:** 2026-06-08

---

We got one goal today: understand one of the common failure modes of Recursive Language Models, and a simple cool way to reduce it.

Recursive Language Models (RLMs) let a model answer questions over a context far larger than its window by treating the prompt as a variable inside a Python REPL.

The agent writes code to search, slice, and chunk the text, and can recursively spawn sub-agents over the pieces. The subagent answers come back inside the REPL values, i.e. the responses are never auto-dumped into the parent's context directly.

Spawning swarms of subagents to divide and conquer a task is super cool! But RLM traces make or break depending on its capability of prompting subagents correctly about: (a) what it should do, (b) what it should return back, and (c) how the main agent aggregates the subagent responses into a final response.

## The NarrativeQA / Saltram Problem

Context: the full ~107K-character text of Henry James's "The Coxon Fund."
Question: "What is Saltram's living situation?"
Correct answer: "He is a guest in the home of the Mulvilles."

The truth is never said outright. Frank Saltram is the permanent house-guest ("inmate") of the Mulvilles at their Wimbledon home. This fact you only get by reading and connecting — not by keyword lookup.

## Depth-0 Solution (No Subagents)

The model searches and slices contexts around key data points and reads relevant sections. Cost: ~$0.04 with Minimax M3. Works correctly.

## Subagents v1: Free-text fan-out (FAILURE)

Split ~107K characters into fixed-size chunks, fanned out 62 free-text sub-agents in parallel, then asked one more to aggregate. Result: subagents returned variations of "The passage does NOT contain direct verbatim quotes..." and the aggregator got overwhelmed. The root agent eventually hand-wrote a wrong answer hallucinating a "set of chambers" and "shabby-genteel part of London."

## Subagents v2: Structured outputs routing (SUCCESS)

Instead of asking sub-agents for prose, asked each a True/False question with a JSON-Schema-constrained answer (`schema={"type": "boolean"}`), then read only the chunks that said True. The booleans acted as a direct attention mask to the original context — external sparsification of the input prompt.

Result: correct answer, same cost (~$0.04). Much reduced hallucination scope.

## How fast-rlm implements structured output validation

1. **Schema normalization** — Accepts Pydantic model, primitive type, list generic, or raw JSON Schema dict. Converts to plain JSON Schema.
2. **Contract displayed at REPL startup** — Model knows exact shape before writing any code.
3. **Validate on every FINAL** — Before returning subagent result, validate against schema. On failure: send feedback with specific errors and expected format.
4. **Retry, don't restart** — REPL work is untouched. Model just fixes the value and re-calls FINAL.

## Key Insight

Structured outputs let the main agent use subagent responses directly inside the REPL as clean, validated data. Instead of parsing 40 variations of "No mention of Saltram's living condition in this paragraph," it reads one boolean flag. Lower hallucinations because the model never had to read too much story into its context at once.
