# Every Agentic Engineering Hack I Know (June 2026)

Three months ago Matt posted "Every Claude Code Hack I Know" — 913K views. This is the sequel, now broadened to Agentic Engineering. He shipped last30days (27K stars), Printing Press (4K+ stars), Agent Cookie, and became a top contributor to CPython and Go — all through agentic workflows.

## The 22 Hacks

**1. The Moment You Have an Idea, Make a CE plan.md**
Before anything else, create a plan.md. The act of writing forces clarity. Doesn't need to be perfect — the agent will help refine it.

**2. Don't Read the plan.md**
After you've written it, don't read it yourself. Let the agent read and execute it. Reading it creates bias about what's possible. Trust the system.

**3. Use /ce-plan for Your Deepest Non-Engineering Work**
Plan.md isn't just for code. Use it for strategy, writing, research, life decisions. Make a plan for the plan.

**4. Get Voice-Pilled**
Voice dictation is faster than typing for ideation. Speak your plan. The agent transcribes and executes. Matt uses it for everything.

**5. Lots and Lots of Tabs in cmux**
Run multiple Claude/Codex sessions in parallel via cmux tabs. Different agents on different tasks simultaneously. One agent writes tests while another refactors.

**6. Make Your Terminal Default Into Claude or Codex, Not a Shell**
Your default terminal should open into an agent, not bash/zsh. The agent becomes your OS interface.

**7. Remote Control Every Window, and Give Claude Code or Codex an Email Address**
Give the agent access to your browser, email, and desktop. It can send emails, browse, and interact with the world on your behalf.

**8. Dangerously Skip Permissions, and Yes I Mean It**
Turn off permission prompts. The friction of approving every tool call kills flow. Trust the agent or don't use it — but don't half-trust it.

**9. How I Run Most of My Code Through Codex Without Ever Opening the Codex CLI**
Use Claude Code as the orchestrator. It delegates to Codex for execution. You never touch Codex directly.

**10. Research Before You Plan: last30days**
Use his last30days tool (27K GitHub stars) to see what's trending in open source before deciding what to build. Data-driven planning.

**11. Granola All the Things, and Put the RAW Transcript In Your LLM**
Record everything — meetings, thoughts, conversations. Feed raw transcripts directly into the agent. Don't summarize first; let the agent extract what matters.

**12. Human Signal**
Insert explicit human checkpoints in agent workflows. The agent proposes, you approve. Keeps you in the loop without micromanaging.

**13. HyperFrames for Video, for All the Things**
Generate video documentation automatically. Feed code + context into video generation. Visual explanations compound.

**14. Your Notes Are Your Agent's Knowledge Base**
Feed your notes (Obsidian, etc.) directly to the agent. Years of accumulated thinking become instantly queryable context.

**15. Work From Anywhere - My Mac mini**
Use a dedicated Mac mini as an always-on agent server. SSH in from anywhere. The agent is always running.

**16. Proof: For Sending a Plan to a Colleague**
Share plan.md files directly with teammates instead of meetings. Async collaboration through structured plans.

**17. Write Your Own Skills**
Don't just use built-in skills. Create custom skills that encode your specific workflows. Skills are the compound interest of agentic engineering.

**18. Open Source: Contribute to the Projects You Love**
Use agents to contribute to major open source projects (CPython, Go). Agents handle the boilerplate; you provide the insight.

**19. My Current Laptop Setup**
Detailed hardware/software stack optimized for agentic workflows.

**20. Printing Press: CLIs That Run Real Life**
His tool for creating CLIs that automate real-world tasks. Agents that interact with physical-world services.

**21. The Honest Part: AI Psychosis**
Agents sometimes go off the rails — creating files you didn't ask for, making changes that don't make sense. It's part of the process. Learn to recognize and recover.

**22. This Article Was Written This Way**
Meta: this very article was created using the techniques described in it. Plan.md → voice dictation → agent execution → human signal checkpoints.
