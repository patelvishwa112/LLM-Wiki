The 5-stage memory progression.\nThe single most useful framing for what “agent memory” means in practice comes from the Anthropic team’s Continual Learning Bench 1.0 experiment. Effective use of memory requires a progression of five stages. Each stage is a structural move; each model exits the progression at a different point.\n1. Fail - the agent gets something wrong and documents the failure with enough detail to be useful later.\n2. Investigate — before moving on, the agent figures out why the failure happened.\n3. Verify - the agent turns the diagnosis into a checked fact, not a guess.\n4. Distill - the agent turns the verification into a general rule that applies beyond the specific case.\n5. Consult - on the next task, the agent reads the rule instead of re-deriving the fact from scratch.\nThe measured difference between models on a SQL exploration task from the Continual Learning Bench, each model with memory provided:\nSonnet 4.6 exits at step 1. Its memory store is a list of failure notes and open guesses (“maybe prc instead of prc_usd?”). It rarely consults prior notes. Memory exists but doesn’t compound.\nOpus 4.7 exits at step 3. It creates a schema reference with uncertainty flagged (“possibly prc in cents? Verify.”). Verification coverage runs 7–33% (median ~17%) of questions.\nFable 5 tends to complete the progression. In its strongest runs, verification coverage reaches 73% (22 of 30), and it distills learnings into general rules that help with future tasks.\n\n\n11. The state file.
