## Content

Here is the full timeline of policy gradients from REINFORCE to PPO (and GRPO):

- REINFORCE: you collect trajectories in the environment. Calculate the returns at each stage using the rewards you saw during rollout. Increase the probability of actions that appeared in high-return trajectories. 

This is one of the purest forms of policy gradient because it directly uses Monte Carlo rollout returns. Because of how uncertain rollouts can be in certain environments, REINFORCE has high variance (one of it's main drawbacks). 

REINFORCE is unbiased because it does not require a value/critic network, although you can add a baseline to reduce variance without biasing the gradient, as long as the baseline does not depend on the action. Baselines are similar ideas to what people refer to as Advantage today.

- Actor Critic: In REINFORCE, you have to wait till the end of the a full episode to compute full returns, and then learn.
In Actor Critic, you instead have
1. critic: a value function that computes scalar quality value of being in a state
2. an actor: learns probability of actions given the estimates of the critic.

REINFORCE has high "variance" coz different rollouts can have different rewards. AC lowers variance coz the actor learns from the critic scores instead of the raw rollouts. This is also why AC is a "biased estimator" coz we are using the critic as a proxy for the real rollout scores.

Also google: MC (Monte Carlo Sampling) vs 1-step TD (Temporal Difference Learning).

- A2C/A3C: Previous AC used raw returns to calculate value network scores. They learn if the "return is high" in a state. But raw return can be misleading.

Suppose one state is naturally high-value. Almost every action there gets good returns. Another state is naturally low-value. Even good actions there may get low returns. If we just looked at raw returns we would end up lowering probabilities of good actions in bad states too.

Advantage measures how much better or worse an action is compared with the average action you would normally take in that state. I.e. it is a relative metric.

Basically if Advantage of action A in state S is greater than 0, means the action was better than average for that state. 

Around this time, people also started using Entropy bonuses in the loss function to encourage exploration

- TRPO: Trust Region Policy Optimization
TRPO is largely a stability idea. The goal of TRPO is to create a "trust region" for the network so that it does not freeform update itself. Instead a network is allowed to update only within certain constraints.

Different algorithms decide how these constraints are created. 

- PPO: Proximal Policy Optimization
PPO keeps the central idea of TRPO (avoid large destructive policy updates) and makes the math and implentation much simpler. Google: "clipped surrogate objective in PPO to learn more"

Basically, the clipping prevents the new policy from moving too far from the old one.

Advantage also got upgraded with GAE (Generalized Advantage Estimate). Here you can control the bias-variance tradeoff using a seperate hyperparameter. High variance means that we use the actual rollout scores (REINFORCE style), and high bias means we use the critic network (A2C style).

- GRPO: It is a variant of PPO where instead of training a seperate critic network.

Instead, they sample multiple completions for the same prompt and compare them against each other. Then compute a relative advantage within the group. 

GRPO still has the same PPO clipped-surrogate objective.

PPO still uses the Actor framework from REINFROCE.

REINFORCE is derived from the policy gradient theorem.
