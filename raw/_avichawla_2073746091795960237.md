## Post

Stanford researchers built the agent-native version of Git.

When an agent works on a longer task, the run builds up state: files edited/created, dev server, database, installed packages, KV cache, etc.

At step 10 a mistake can derail a run even when steps 1–8 were correct. Default recovery is fix-forward (more tokens, more context) or restart from step one (re-pay calls, non-deterministic replay).

Message logs record tool calls but not live state: memory, handles, child processes, packages, /tmp, KV cache. Git versions files but not the running process or warm cache.

**Shepherd** records runs as a trace of typed events. Each agent–environment interaction is a commit (agent process + filesystem, copy-on-write). Forking from a commit restores actual state. ~5× faster than docker commit; >95% KV cache reuse on unchanged prefix.

A meta-agent can watch the trace and revert before bad writes commit. Not everything is reversible (external side effects need prevention).

**CooperBench** (two agents, same codebase): live supervisor raised pass rate **28.8% → 54.7%**. Alpha; biggest wins on long, heavily branched sandbox runs.
