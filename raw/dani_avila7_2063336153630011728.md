How NVIDIA's SkillSpector works and How to add It to your skill PRs

AI agents run code they didn't write. Skills, tools, MCP servers, prompts pulled from external repos. The attack surface is real, and most teams aren't thinking about it yet.

NVIDIA's SkillSpector is their answer, and this is how you run it.

What it is

SkillSpector is a static security scanner built for AI agent skills (Claude Code, Codex, Antigravity, Cursor, and similar). It answers one question: is this skill safe to install?

Repo: https://github.com/nvidia/skillspector

It analyzes the SKILL.md and every script in the folder, runs 64 detection patterns across 16 categories, and produces a risk score from 0 to 100 with a clear verdict. Apache-2.0, so you can read exactly what it's doing.

How it works

The scanner builds a context from the skill folder, then runs analyzers in parallel:

• Prompt injection and instruction overrides
• Data exfiltration (env var harvesting, context leakage, external transmission)
• Supply chain (curl | bash, CVEs via OSV.dev, typosquatting)
• Behavioral AST (eval, exec, subprocess, dynamic imports)
• Taint tracking (credentials flowing to the network, file read to exfiltration)
• YARA signatures for malware, webshells, cryptominers
• MCP-specific patterns: tool poisoning, unicode deception, least-privilege violations

Scoring is additive: LOW +5, MEDIUM +10, HIGH +25, CRITICAL +50. A single CRITICAL pattern puts you at 50 before anything else is counted.

If executable scripts are present, the total is multiplied by 1.3. The final number maps to a verdict on a 0–100 scale, anything above 50 is blocked automatically.

Reports come out as terminal output, JSON, Markdown, or SARIF, which is what makes it drop into CI cleanly.

Adding it to your Skill PRs

SkillSpector plugs into GitHub Actions without much ceremony.

On every PR that touches your skills directory, only the changed skills are scanned, fast, deterministic, no API keys needed in --no-llm mode.

The scan posts a Markdown report directly on the PR, uploads SARIF to the Security tab, and blocks the merge if any skill scores above 50. A separate weekly workflow scans the full registry for ongoing visibility but never blocks. The gate only lives on PRs.

Seen in practice: Claude Code Templates

Claude Code Templates (aitmpl.com) runs SkillSpector on every PR that adds or modifies a skill in the registry.

Check this PR: https://github.com/davila7/claude-code-templates/pull/623

A synthetic malicious skill submitted for testing scored 84/100 CRITICAL and was blocked before merge.

If you want to see the scanner in action, submit a skill at https://github.com/davila7/claude-code-templates and watch the security report appear directly on your PR.

Why it matters now

The Skills ecosystem is growing faster than security practices are catching up. Most developers installing a community skill have no idea what scripts it runs or where it sends data.

SkillSpector makes that visible before anything executes.
