# A Brain in a Git Repo: How GBrain Works

Mem0 (@mem0ai) — In Context #14

GBrain is an open-source brain for AI agents, built by @garrytan (Y Combinator), open-sourced April 2026. Knowledge as markdown in a git repo you own, then indexed for retrieval. Garry runs it behind OpenClaw and Hermes agents; self-reported scale: 146,646 pages, 24,585 people, 5,339 companies, 66 overnight cron jobs.

Design highlight: knowledge graph from pattern matching (wikilinks, typed links) with **zero LLM calls at graph build time** — largest driver of retrieval quality per BrainBench.

## Markdown is the source of truth

Brain repo is authoritative; Postgres/PGLite index is downstream. Soft-delete on sync when git removes files. Diffable, PR-reviewable, git revert for bad writes. Engines: PGLite (WASM Postgres, ~50k pages personal) or Postgres+pgvector at scale; `BrainEngine` ~47 ops, CLI/MCP generated from one contract.

## The self-wiring graph

`put_page` extracts entities from `[[wiki/people/bob]]` and typed-link syntax via regex → `links` table (`from_page_id`, `to_page_id`, `link_type`, `context`). Types: attended, works_at, invested_in, founded, advises, mentions. Multi-hop via recursive SQL / `gbrain graph-query`.

BrainBench v0.20.0 (author's fictional 240-page corpus, 145 relational questions):
- Full system: P@5 49.1%, R@5 97.9%
- Graph disabled (vector+keyword): P@5 17.8%
- Vector only: P@5 10.8%
- "Graph layer worth 31 points P@5"

Zero LLM = write-time graph only; retrieval/synthesis still uses tokens.

## Hybrid retrieval and synthesis

- `gbrain search` — hybrid HNSW vector (text-embedding-3-large 1536d) + tsvector keyword + pg_trgm fuzzy titles, RRF `1/(60+rank)`, Haiku query expansion, dedup layers.
- `gbrain think` — synthesized answer with citations **and** explicit gaps (stale, uncited, contradictions, holes).

## Dream cycle

Overnight cron: dedup people, fix citations, score salience, contradictions, prep tasks. Reference 20+ jobs; Tan runs 66. `gbrain eval suspected-contradictions` with LLM judge in daily dream. Shell jobs (v0.14+) for deterministic cron — zero tokens, ~60% gateway headroom reclaimed.

## Brain is not memory

Brain = world knowledge (people, companies, deals, concepts). Memory = preferences, tool config, session continuity. Critical world facts must live in git-durable brain; GBrain docs say operational state belongs in **agent memory** layer (Mem0 positioned here).

## Caveats

BrainBench is self-administered, not comparable across vendors; production page counts are self-report. Regex graph needs disciplined wikilinks. PGLite single-writer contends with MCP+cron. Cloudflare (Apr 2026) argues managed pipelines beat raw filesystem for agents.

## Complementary stack

GBrain: relational world knowledge, files you own. Mem0: meaning-based, identity-scoped memory across tools at 1M–10M token scale. Article links prior **State of Memory in Agent Harness** (mem0, Jun 2).
