On June 2 2026, Anthropic released the ant CLI. One binary, written in Go, MIT licensed, open source. Every Claude API endpoint accessible from terminal: messages, models, files, batch jobs, agents, sessions, environments, skills. All of it.

This is not a wrapper. This is the operational layer for a platform shifting from model provider to full agent infrastructure.

## What ant actually is

ant is a typed API client. You pass flags → it builds the request → handles auth → paginates automatically → formats the response based on context. Single binary, zero runtime dependencies.

Code lives at: github.com/anthropics/anthropic-cli

Command structure is consistent: `ant {resource} {action}`, colons for nested resources:
```
ant models list
ant messages create --model claude-opus-4-8 --max-tokens 1024
ant beta:agents retrieve --agent-id agent_01...
ant beta:sessions:events list --session-id session_01...
```

Beta resources (agents, sessions, deployments, environments, skills, vaults, memory stores) all live under `beta:` prefix. The CLI automatically sends `anthropic-beta: managed-agents-2026-04-01` header for every beta command.

## Installation

```bash
# macOS via Homebrew
brew install anthropics/tap/ant

# Linux or WSL
curl -fsSL https://console.anthropic.com/install/cli | sh

# Via Go
go install github.com/anthropics/ant-cli@latest

# Verify
ant --version
```

Environment variable `ANTHROPIC_API_KEY` wins over named profiles. If commands hit wrong workspace, check that first.

## Output Formats

- `--format yaml` — easier to scan than JSON
- `--format jsonl` — one object per line, built for pipelines
- `--format explore` — interactive TUI explorer with arrow key navigation, `/` to search, `q` to quit

`--transform` flag takes a GJSON path and reshapes the response:
```bash
ant beta:agents list --transform "{id,name,model}" --format jsonl
```
Pair with `--raw-output` when you need a resource ID for the next command.

**@path references** — inline local file content directly. Plain text goes as string literal. Binary files (images, PDFs) get base64-encoded automatically. `@file://path` for plain text, `@data://path` for base64.

## Managed Agents — Full Workflow

### The Four Primitives

- **Agent:** configuration (model, system prompt, tools, MCP servers). Create once, reference by ID.
- **Environment:** where agent runs. Cloud container with networking, sandboxing, pre-installed packages.
- **Session:** running instance of agent inside environment. Durable — container crash → fresh instance reads append-only event log → resumes.
- **Events:** protocol. User messages, tool results, status updates flow as SSE. `session.status_idle` when done.

### Step 1: Create Agent
```bash
ant beta:agents create \
  --name "Code Reviewer" \
  --model '{id: claude-opus-4-8}' \
  --system "You are a senior code reviewer..." \
  --tool '{type: agent_toolset_20260401}'
```
Agents can also be defined as YAML files checked into Git:
```yaml
name: Code Reviewer
model:
  id: claude-opus-4-8
system: |
  You are a senior code reviewer.
tools:
  - type: agent_toolset_20260401
```
Deploy: `ant beta:agents create < code-reviewer.yaml`

### Step 2: Create Environment
```bash
ant beta:environments create \
  --name "reviewer-env" \
  --config '{type: cloud, networking: {type: unrestricted}}'
```

### Step 3: Start Session
```bash
ant beta:sessions create \
  --agent '{type: agent, id: agent_01..., version: 1}' \
  --environment-id env_01... \
  --title "Review my utils module"
```
Sessions cost $0.08 per session-hour, billed to the millisecond. Idle time is free.

### Step 4: Send Task
```bash
ant beta:sessions:events send \
  --session-id sesn_01... \
  --event '{type: user.message, content: [{type: text, text: "Review this..."}]}'
```

Pull event trace: `ant beta:sessions:events list --session-id sesn_01... --format yaml`

## Three Features That Shipped May 6 2026

### Dreaming (research preview)
Scheduled background process between sessions. Reviews past sessions, extracts patterns, curates memories. Harvey (legal AI) saw ~6x task completion rate increase.

### Outcomes (public beta)
Self-grading evaluator loop. Separate evaluator scores output against written rubric → tells agent what to fix → agent retries → up to 20 iterations. Wisedocs cut document review time by 50%.

### Multi-agent Orchestration (public beta)
Lead agent breaks job into pieces → delegates to specialist sub-agents → runs in parallel → merges results. Netflix processing hundreds of builds simultaneously.

## Self-Hosted Sandboxes & MCP Tunnels (May 19)

### Self-hosted sandboxes (public beta)
Tool execution in your infrastructure. Your containers, filesystem, network, audit logs. Supported providers: Cloudflare, Daytona, Modal, Vercel.

Worker process: claims sessions from Anthropic's queue → spawns isolated execution context → downloads agent skills → runs tool calls inside your environment → posts results back.

```bash
ant beta:worker poll   # Always-on, long-running process
ant beta:worker run    # Handle single work item
```

**Important:** image pinning is your responsibility. Tag images by SHA digest. Not yet available on Claude Platform on AWS. Memory not supported in self-hosted mode.

### MCP Tunnels (research preview)
Lightweight gateway inside private network → single outbound HTTPS connection. Cloud-hosted agent can reach internal databases/APIs/services through encrypted channel without exposing anything publicly. Access by request.

## Agents as Code (CI/CD)

Agent definitions are YAML files → version-controlled alongside application code. Example GitHub Actions workflow: checkout → install ant CLI → `ant beta:agents update --agent-id ... < agents/reviewer.yaml`.

## Production Shell Pipelines

```bash
# Find agents on deprecated models
ant beta:agents list --transform "{id,name,model}" --format jsonl | grep "opus-4-1"

# Pull error events from a session
ant beta:sessions:events list --session-id "$SESSION_ID" --transform "{type,content}" --format jsonl | grep '"error"'

# Check token usage across sessions
ant beta:sessions list --transform "{id,title,usage}" --format jsonl
```

`--debug` flag prints full HTTP request/response to stderr without touching stdout.

## Agent Version Management

Every update creates new version number. Version acts as optimistic lock. Pin sessions to specific versions for reproducible deployments:
```bash
ant beta:agents:versions list --agent-id "$AGENT_ID"
ant beta:sessions create --agent '{type: agent, id: "...", version: 3}' ...
```

## ant vs SDK vs Claude Code

- **ant CLI:** operational work — provisioning agents, rotating credentials, monitoring sessions, deploying configs, debugging from terminal, shell pipelines
- **SDK:** application integration — embedding agent interactions in web apps, Slack bots, data pipelines
- **Claude Code:** interactive orchestration — iterating on prompts, reviewing traces, orchestrating cloud agents

**Recommended path:** Prototype with Agent SDK → graduate to Managed Agents in cloud → manage with ant and Claude Code.
