# 6 MCP servers, 0 calls, $280K wasted - and the model was never the problem

**Author:** @dunik_7 (dunik)
**Source:** https://x.com/dunik_7/status/2064342854051209338
**Date:** 2026-06-09

---

$280,000 a year. That's what a 5-dev team quietly burns running Claude Code with no harness around it. Same model. Same subscription. Same prompts. The difference is everything you build around it.

Most devs install the model, type "build me an app," watch it break, and decide AI coding doesn't work. I ran one agent across 5 microservices in production for 3 months. Here is the exact harness every file, every gate, copy-paste.

## The Model Didn't Get Smarter — The Harness Did

80,000 GitHub stars on a "memory" tool that bloated to 700MB and pinned 5 CPU cores doing nothing. 6 MCP servers installed, 0 calls across 98 sessions. One tool — Playwright — called hundreds of times.

A bare model in a chat window is a job interview. A model wrapped in a harness is an employee who's been on the team a quarter. The model is the same in both. What changed is that I gave it a desk, a filing cabinet, a notebook it actually reads, and coworkers it can hand work to.

## 1/4 — THE MAP

Files the agent reads before it writes anything. CLAUDE.md (layered: personal → project → path rules), PRODUCT.md, DESIGN.md, CONTEXT.md, docs/agent-docs/. The agent always has the most specific instruction for where it's actually working.

CLAUDE.md skeleton: Stack, hard rules (no abstractions not asked for, only touch task files, no dependencies without asking), where things go. Mistake: don't write it once — every time the agent does something wrong, add one line. His had 30+ rules by launch.

Cost without map: ~$375/week/dev re-explaining stack.

## 2/4 — THE MEMORY

Two circuits. Circuit one: task dependency graph (he uses `beads` — 1,006 tasks: 55 closed, 50 open, 48 ready). Circuit two: written paper trail — 162 plans, 157 specs, 48 reviews, 15 ADRs. "The documentation IS the memory. The memory IS the moat."

Files: MEMORY.md (decision log), ERRORS.md (failure log — when approach takes 2+ attempts), docs/adr/ (architecture decisions). Mistake: don't keep decisions in your head or Slack. Agent reads neither.

Cost without memory: ~$150/week/dev redoing ruled-out approaches.

## 3/4 — THE HANDS

Usage across 98 sessions: Playwright (hundreds), codegraph (28), context7 (rare). Six servers: ZERO calls. Not "low usage" — connected because a thread said to, then left hanging.

Every connected tool injects its definition into the context window on every single request. 6,000 tokens × 200 req/day × 30 days = 36M tokens/month = ~$108/month paid to make the agent slower and dumber.

The worst offender: a "memory" plugin with 80K+ stars ballooned to 700MB and pinned 5 CPU cores. Tool stars ≠ works in your harness.

Audit prompt included — sort every tool into KEEP/WATCH/CUT, default to cutting. Mistake: don't add tools "to be safe." Start with nothing.

## 4/4 — THE TEAM

Sub-agent breakdown: code-review (36), backend (31), explore/general (31), frontend/fullstack (13/7), architect/security (3/3). More work went to sub-agents than I did directly. Each gets fresh, clean context.

Review gate: separate agent reviews diff + original task. Builder CANNOT approve own work. Hooks: session start → load git + restore tasks, pre-compaction → save context, post-edit → auto-lint, locked folders → hard block, task completion → pytest+tsc (red blocks completion).

Flow: load context → brainstorm → plan → delegate to sub-agents → TDD loop → commit → independent review → close.

Cost without team/gate: ~$450/week/dev reverting out-of-scope changes + debugging tired context.

## The Math

- ~$108/week dead tools
- ~$375/week no map
- ~$150/week no memory  
- ~$450/week no team/gate
- **≈ $1,083/week per developer**
- **Team of 5: ~$280,000/year**

## How to Start

Start with the tool audit. 10 minutes. Cut everything with zero calls. Add the map next, then memory files, then the review gate. One layer a week, as you hit each wall.
