# Four-layer agent memory stack (Ella/Lyra)

**Author:** Matt Gunnin (@MatthewGunnin)  
**Posted:** 2026-07-02  
**Engagement:** 102 likes, 14 reposts

TLDR: Most agents reset and forget. Here's the exact memory stack I've built so my agents share context across restarts, coordinate without stepping on each other, and retain decisions for months.

## Problem

Typical setup: cram system prompt, hope context holds, restart loses everything. Author runs **Ella (OpenClaw)** and **Lyra (Hermes)** for ~6 months—memory architecture matters more than model/tools.

## Layer 1: In-session context

Session start reads **identity file** (editable markdown behavior/priorities) and **memory index** (TOC to per-fact markdown files; index always loaded, files on demand). Debuggable without API/dashboard.

## Layer 2: Post-session retention (Hindsight)

Local facts backend (localhost); end of session pushes curated facts to per-agent banks: decisions, inferred facts, failure/fix patterns, confirmed preferences. New sessions query Hindsight before answering. Promotion path: Hindsight fact → human review → memory index entry.

## Layer 3: Shared long-term state (Nexus)

Obsidian vault: live-context append-only log (read before reply, append after meaningful turn), project-state, decisions log, per-agent working-context checkpoints. Cross-agent handshake without message bus.

## Layer 4: Searchable knowledge (gbrain)

MCP semantic/full-text search over Nexus—ranked pages with provenance; episodic memory (layers 1–3) vs lookup recall.

## Cross-agent invariant

Signed append-only entries (agent, channel, kind, summary); acknowledge peer entries; significant decisions duplicated to decisions log with rationale. One race-condition reconciliation anecdote.

## Tradeoff

Requires discipline—write, maintain, review retention; not magic always-on.

## How to start

1. Identity + memory index at session start; write repeat corrections to index.
2. Shared live-context file for multi-window/multi-agent.
3. Manual retention until pattern trusted, then automate flagging.
4. One file per fact via index pointers.

Layers 1+3 ~ one weekend; full stack ~6 months to stabilize.
